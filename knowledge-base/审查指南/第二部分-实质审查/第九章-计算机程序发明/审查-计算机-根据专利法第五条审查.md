# 审查-计算机-根据专利法第五条审查

> **来源：** 《专利审查指南》第二部分第九章 §6.1.1
> **对应法条：** 《专利法》第五条第一款
> **最新修订：** 2023年修订（新增章节）
> **关联页面：** [[审查-计算机-审查基准]]、[[审查-不授权-违反法律与公序良俗]]、[[客体-原理-道德审查]]、[[不授予专利权的客体-违反法律与社会公德]]

---

## 核心要点

2023年修订新增了"根据专利法第五条第一款的审查"章节（§6.1.1），专门规定涉及人工智能、大数据等领域发明的违法性审查。核心内容是：如果发明的数据采集、标签管理、推荐决策等功能违反法律法规或妨害公共利益，则根据专利法第五条第一款不能被授予专利权。新增了商场床垫人脸识别示例和无人驾驶应急决策示例两个具体案例。

---

## 审查标准

### 1. 审查原则

涉及人工智能、大数据等的发明专利申请，如果其实施会违反法律、社会公德或者妨害公共利益，则根据专利法第五条第一款的规定不能被授予专利权。

**专利法第五条第一款的立法目的**：专利法第五条第一款旨在排除那些实施后会损害社会公共利益、违反社会基本伦理或破坏法律秩序的发明创造获得专利权。对于涉及计算机程序的发明，特别是人工智能、大数据等新兴技术领域的发明，其技术方案的实施往往涉及数据处理、算法决策等，这些活动可能触及隐私保护、数据安全、社会公平等敏感领域，因此需要特别关注其合法性和合伦理性。

#### 1.1 "违反法律"的判断标准

"违反法律"是指发明创造的实施违反由全国人民代表大会或全国人民代表大会常务委员会制定并公布实施的法律，不包括仅其实施为法律所禁止的发明创造。例如，一种吸毒工具的制造方法本身属于违反法律的发明，但一种具有合法医疗用途的药物的制造方法，即使该药物可能被滥用，其本身并不违反法律。

在AI/大数据领域，"违反法律"通常涉及以下法律法规：
- 《个人信息保护法》
- 《数据安全法》
- 《网络安全法》
- 《民法典》中关于隐私权和个人信息保护的规定
- 其他与数据采集、处理、使用相关的专门法律

#### 1.2 "违反社会公德"的判断标准

"社会公德"是指公众普遍认同并遵守的道德准则和行为规范。在AI/大数据领域，违反社会公德通常涉及：
- 基于种族、性别、宗教信仰等进行歧视性处理；
- 将人按照经济价值、社会地位等进行不平等的排序或决策；
- 实施方式严重违背社会公认的基本伦理准则。

#### 1.3 "妨害公共利益"的判断标准

"妨害公共利益"是指发明创造的实施会对不特定多数人的共同利益造成损害。在AI/大数据领域，常见情形包括：
- 大规模未经授权的数据采集威胁公众隐私安全；
- 算法决策导致社会资源配置严重不公；
- 技术方案的滥用可能危害公共安全。

### 2. 重点审查领域

#### 2.1 数据采集的合法性

如果发明涉及的数据采集方式违反法律法规，例如未经授权采集个人隐私数据（人脸识别数据、位置数据、通信数据等），则该发明不能被授予专利权。

**【商场床垫人脸识别示例】**

一种商场用床垫推荐系统，在消费者不知情的情况下，通过设置在商场床垫上的摄像头采集消费者的人脸图像，分析消费者的面部特征以推测其消费偏好，从而进行床垫推荐。由于该方法在消费者不知情的情况下采集人脸图像，侵犯了消费者的隐私权，违反了相关法律法规，因此该发明违反专利法第五条第一款的规定，不能被授予专利权。

**审查要点分析**：
- **采集方式**：在消费者"不知情"的情况下采集；
- **采集对象**：人脸图像（属于敏感个人信息）；
- **违法性**：违反《个人信息保护法》关于告知同意原则的规定；
- **结论**：违反专利法第五条第一款。

**AI/大数据领域数据采集违法的审查边界**：

| 采集情形 | 是否可能违反第五条 | 说明 |
|---------|------------------|------|
| 未经授权采集人脸识别数据 | 是 | 人脸信息属于敏感个人信息，需单独同意 |
| 未经授权采集位置轨迹数据 | 是 | 位置信息可识别特定自然人，属于个人信息 |
| 未经授权采集通信内容 | 是 | 通信秘密受宪法和法律保护 |
| 在获得用户明示同意下采集数据 | 否 | 符合合法性基础 |
| 采集已合法公开的个人信息 | 否 | 但需在合理范围内处理 |
| 通过爬虫技术抓取公开网络数据 | 一般否 | 但若违反网站Robots协议或破坏技术措施，可能违法 |
| 通过黑客手段非法获取数据 | 是 | 属于违法行为 |

#### 2.2 数据标签管理的合法性

如果发明涉及的数据标签管理方式违反社会公德或妨害公共利益，例如基于种族、性别、宗教信仰等进行歧视性标签管理，则该发明不能被授予专利权。

**审查要点**：
- 标签分类是否基于法律法规禁止的歧视性标准；
- 标签管理是否会导致对特定群体的不公平对待；
- 标签的使用是否会造成社会分裂或冲突。

**典型不合规情形**：
- 基于种族、肤色进行用户信用评分；
- 基于性别进行招聘筛选；
- 基于宗教信仰进行服务推荐限制。

#### 2.3 推荐决策的合法性

如果发明涉及的推荐决策机制违反法律法规，例如在无人驾驶等涉及生命安全的场景中采用违法的应急决策规则，则该发明不能被授予专利权。

**【无人驾驶应急决策示例】**

一种无人驾驶车辆在紧急情况下的决策方法，该方法在面临不可避免的碰撞时，根据行人或车辆的经济价值（如车辆价格、行人的社会地位等）来决定碰撞对象。这种基于经济价值或社会地位来决定生命安全优先级的决策方法违反了社会公德，妨害公共利益，因此该发明违反专利法第五条第一款的规定，不能被授予专利权。

**审查要点分析**：
- **决策依据**：以"经济价值""社会地位"决定碰撞对象；
- **伦理问题**：将人的生命安全货币化、等级化；
- **违反的规范**：社会公德（生命平等原则）；
- **结论**：违反专利法第五条第一款。

### 3. 人脸识别等敏感技术的审查边界

人脸识别技术作为生物识别技术的一种，因其具有唯一性、不可更改性等特点，在审查中需要特别关注其合规性。

#### 3.1 人脸识别技术的特殊审查规则

| 应用场景 | 合规要求 | 常见违法情形 |
|---------|---------|-------------|
| 公共场所安防 | 需符合公共安全目的，设置显著标识 | 未告知即采集；超范围使用 |
| 商业营销 | 原则上需取得个人单独同意 | 未经同意采集；隐性采集 |
| 金融服务 | 需符合金融监管要求，明示目的 | 强制授权；过度采集 |
| 门禁考勤 | 需告知并取得同意，提供替代方式 | 唯一验证方式；数据泄露 |
| 教育教学 | 需符合未成年人保护规定 | 未经监护人同意采集未成年人面部信息 |

#### 3.2 敏感技术审查的一般原则

对于涉及人脸识别、基因数据、健康医疗数据等敏感技术的发明专利申请：
- **目的限定原则**：数据采集和处理的目的应当明确、合法，不得超出目的范围使用；
- **最小必要原则**：采集的数据类型和范围应当与实现目的直接相关，不得过度采集；
- **知情同意原则**：涉及个人信息的采集，应当告知并取得同意（法律法规另有规定的除外）；
- **安全保障原则**：应当采取必要的技术措施保障数据安全。

### 4. 更多审查示例

**【示例3】基于种族歧视的信用评估系统**

一种个人信用评估方法，通过分析用户的种族背景、肤色特征等生物特征数据，结合消费记录，预测用户的信用风险等级。该方法将种族作为信用评估的因素之一，可能导致对特定种族群体的不公平对待，违反社会公德，不能被授予专利权。

**【示例4】非法获取通信数据的网络监控系统**

一种网络监控系统，通过在公共WiFi网络中植入嗅探程序，截获并分析用户的通信数据包，以推送定向广告。该系统在未获得用户同意的情况下截获通信数据，违反了通信秘密保护的相关法律规定，不能被授予专利权。

**【示例5】合法授权下的健康数据分析系统**

一种基于可穿戴设备数据的慢性病预警系统，用户在注册时明确授权系统收集其心率、步数、睡眠等健康数据，系统通过机器学习模型分析数据变化趋势，向用户发出健康预警。该系统在获得用户授权的前提下采集和使用数据，数据采集方式合法，不违反专利法第五条第一款。

### 5. 审查注意事项

（1）审查时应当结合说明书的内容，判断发明的实施是否会违反法律、社会公德或妨害公共利益。

（2）如果发明本身的目的并不违法，但说明书中描述的某些实施方式可能违法，应当要求申请人修改或删除相关内容。

（3）对于涉及个人数据处理的发明，应当关注数据采集方式是否符合相关法律法规的规定。

（4）审查时应当区分"发明本身违法"与"发明可能被滥用"：若发明本身的目的和实施方式均合法，仅存在被第三方滥用的可能性，一般不能据此认定违反第五条第一款。

（5）对于涉及公共安全、生命健康等重大利益的发明，应当从严审查其数据采集和决策机制的合法性与合伦理性。

## 实务应对

**代理人视角：**

1. **数据采集合规性审查**：在撰写涉及数据采集的AI发明时，必须在说明书中明确数据的合法获取方式。避免描述未经授权采集个人信息（特别是人脸、位置、通信等敏感数据）的实施方式。建议增加"在获得用户授权/同意的情况下"的限定语。对于涉及人脸识别的发明，应特别强调已获得用户的单独同意。

2. **应急决策的伦理合规**：涉及自动驾驶、医疗AI等涉及生命安全的决策系统，应避免使用基于经济价值、社会地位等因素的决策规则。建议采用基于物理规律、安全距离、伤害最小化等客观技术标准的决策机制。在说明书中应明确记载决策机制的技术合理性，而非价值判断。

3. **算法歧视的规避**：涉及用户画像、推荐系统等发明，应避免基于种族、性别、宗教等敏感属性的歧视性处理。建议在说明书中强调算法的公平性和中立性。若发明涉及对不同群体的差异化处理，应说明该差异化的技术合理性（如基于生理特征的合理差异），而非基于歧视性标准。

4. **说明书的合规修改**：如果审查员指出特定实施方式违反专利法第五条，可以通过删除或修改相关实施例来克服，而不必放弃整个申请。例如，将"采集用户的人脸图像"修改为"在获得用户授权后，采集用户自愿提供的人脸图像"。

5. **区分合法目的与违法手段**：代理人在撰写时应确保发明的目的合法，且为实现该目的所采用的手段也合法。避免在说明书中记载任何可能被视为违法的实施方式，即使这些方式仅是可选实施例。

6. **敏感技术的特别说明**：对于涉及人脸识别、基因数据、健康数据等敏感技术的发明，建议在说明书中增加专门的合规性说明段落，明确指出数据采集和处理符合相关法律法规的要求，以预防审查员的第五条质疑。

## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 未经授权采集人脸识别数据的推荐系统 | 不授予专利权 | 违反隐私保护法律，专利法第五条第一款 |
| 基于经济价值/社会地位决定碰撞对象的无人驾驶应急决策 | 不授予专利权 | 违反社会公德，妨害公共利益 |
| 基于种族、性别等进行歧视性标签管理的系统 | 不授予专利权 | 违反社会公德，妨害公共利益 |
| 在用户授权下采集数据进行分析的推荐系统 | 可授予专利权 | 数据采集方式合法 |
| 基于物理规律和安全距离的自动驾驶避障决策 | 可授予专利权 | 符合技术标准和伦理要求 |
| 合法获取公开数据的大数据分析系统 | 可授予专利权 | 数据获取方式合法合规 |
| 通过黑客手段非法获取数据的系统 | 不授予专利权 | 违反网络安全法律 |
| 发明目的合法但某些实施例可能违法 | 要求删除或修改相关实施例 | 专利法第五条第一款 |
| 采集敏感个人信息未取得单独同意 | 不授予专利权 | 违反《个人信息保护法》 |
| 仅存在被第三方滥用可能性的合法发明 | 一般可授予专利权 | 发明本身不违法 |

## 四、动态模板带来的架构性改进

### 4.1 模板机制对比

| 特性 | V1（硬编码） | V2（动态模板） |
|------|------------|--------------|
| 提示词更新 | 需修改源码重新编译 | 修改 `.md` 模板文件即可 |
| 条件段落 | 不支持 | `{{#if}}` 条件渲染 |
| 知识库注入 | 硬编码或跳过 | 模板变量动态注入 |
| 术语标准化 | Agent 内硬编码 60+ 映射 | `shared/terminology.md` 外部维护 |
| 领域指导 | 硬编码 `FIELD_GUIDE_MAP` | 可扩展为独立模板文件 |
| 多语言支持 | 不支持 | 模板文件可按语言创建 |

### 4.2 SkillLoader 降级机制

```
SkillLoader.load('system-prompt')
  → 文件存在 → 渲染模板 → 返回增强提示词
  → 文件不存在 → catch → 降级到硬编码提示词
```

V2 的降级路径保证了即使模板缺失，Agent 仍能正常工作（即 V1 行为）。

## 五、结论

### 质量提升量化

| 指标 | V1 | V2 | 提升 |
|------|----|-----|------|
| 发明三元组置信度均值 | 0.90 | 0.915 | +1.7% |
| 对比分析特征覆盖（权利要求映射） | 0% | 100% | +100% |
| 技术效果量化率 | ~40% | ~85% | +112% |
| 术语规范率 | ~60% | ~95% | +58% |
| 问题纯度（不含解决手段） | ~75% | 100% | +33% |

### 核心结论

1. **术语标准化**是 V2 最显著的改进 — 统一使用"固定连接"、"配置"、"采用"等专利撰写规范用语，为后续权利要求生成提供了更规范的基础数据。

2. **权利要求映射**（对比文件分析）是 V2 新增的关键能力 — 每个特征标注来源权利要求号，使得必要性分类（essential/important/optional）有据可依，不再依赖经验判断。

3. **领域指导注入**使 V2 的特征描述更具体 — 机械工程领域的"形状/位置/连接关系/配合方式"四维描述框架，使技术特征从笼统变为精确。

4. **交叉验证**（发明理解分析）是 V2 新增的质量保障机制 — 确保三元组之间术语一致、逻辑互补，减少了后续撰写中的矛盾风险。

5. **动态模板的维护成本优势** — 后续优化提示词只需编辑 `.md` 文件，无需修改源码和重新编译，显著降低了迭代成本。

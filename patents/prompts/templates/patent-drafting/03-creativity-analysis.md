## 核心分析框架

### 创造性判断"三步法"

**法律依据**: 《专利法》第22条第3款

**核心标准**: 对所属领域技术人员而言，发明相对于现有技术是否**显而易见**

```
创造性判断"三步法"
│
├── 第一步：确定最接近的现有技术
│   ├── 考量技术领域（相同/相近优先）
│   ├── 考量技术问题（应与发明关注的问题有关联）
│   └── 考量发明构思和技术手段（最关键因素）
│
├── 第二步：确定区别特征和实际解决的技术问题
│   ├── 分析发明与最接近现有技术的区别特征
│   └── 根据区别特征的技术效果，认定实际解决的技术问题
│
└── 第三步：判断是否显而易见（技术启示）
    ├── 在现有技术中寻找技术手段
    ├── 分析改进动机
    ├── 把握发明构思
    ├── 准确理解现有技术公开内容
    └── 合乎逻辑的分析推理
```

## 第一步：确定最接近的现有技术

### 1.1 选择标准

**基本规则**: 最接近的现有技术是现有技术中与要求保护的发明最密切相关的一个技术方案

**考量因素**（按优先级排序）:

| 因素         | 说明                             | 优先级     |
| ------------ | -------------------------------- | ---------- |
| **技术领域** | 相同或相近技术领域优先           | ⭐⭐⭐     |
| **技术问题** | 所解决的技术问题与发明相同或相似 | ⭐⭐⭐     |
| **技术手段** | 发明构思与技术手段最相似         | ⭐⭐⭐⭐⭐ |
| **技术效果** | 产生的技术效果相同或相似         | ⭐⭐       |

### 1.2 选择原则

**原则1: 技术领域相同或相近**

- 相同技术领域: 例如"汽车制动系统" vs "汽车制动系统"
- 相近技术领域: 例如"汽车制动系统" vs "轨道交通制动系统"
- 不同技术领域: 例如"汽车制动系统" vs "自行车制动系统"（一般不选）

**原则2: 技术问题相关**

- 所解决的技术问题应当与发明关注的问题有关联
- 不要求完全相同，但应当在同一技术范畴内

**原则3: 发明构思相似（最关键）**

- 发明构思是发明人为了解决技术问题所采用的技术思路
- 即使技术领域不同，如果发明构思相似，也可以作为最接近的现有技术

### 1.3 典型案例

#### 案例: 热管散热器

**现有技术1**: 散热器，使用热管散热，热管与热源接触面是平的
**现有技术2**: CPU散热器，使用风扇散热
**发明**: 热管散热器，热管和基板的接触面设置为相同的平面度

**分析**:

- 技术领域: 发明与现有技术1相同（都是热管散热） ✅
- 技术问题: 都是为了提高散热效率 ✅
- 发明构思: 都是通过热管传导热量 ✅

**结论**: 现有技术1是最接近的现有技术

## 第二步：确定区别特征和实际解决的技术问题

### 2.1 区别特征的确定

**基本方法**: 逐一对比发明与最接近现有技术的技术特征，找出不同之处

**注意事项**:

1. **整体考虑**: 不要将技术方案碎片化
2. **功能对应**: 技术特征应当具有相同或相似的功能
3. **层次对应**: 在同一技术层次上对比

**模板示例**:

```
最接近现有技术: 包括A、B、C、D、E五个技术特征
本发明:           包括A、B、C、F、G五个技术特征

区别特征:
1. 本发明用F替代了现有技术的D
2. 本发明增加了G
3. 本发明省略了E
```

### 2.2 实际解决的技术问题的确定

**基本方法**: 根据区别特征在发明中所起的作用和技术效果，确定发明实际解决的技术问题

**判断流程**:

```
区别特征 → 技术效果 → 技术问题

示例：
区别特征: 将热管和基板设为相同平面度
↓
技术效果: 增大接触面积，提高热传导效率
↓
技术问题: 如何提高热管散热器的散热效率
```

**关键原则**:

1. **以区别特征为依据**: 技术问题应当基于区别特征的技术效果来确定
2. **避免过于上位**: 技术问题不应过于宽泛（如"提供一种更好的方法"）
3. **避免过于下位**: 技术问题不应过于具体（如"将参数设为X"）
4. **与发明目的对应**: 技术问题应当与发明要解决的核心问题一致

### 2.3 典型案例

#### 案例: 自动驾驶掉头方法

**最接近现有技术**: A\*路径规划算法

**区别特征**:

1. 本发明引入"运动基元采样"（A\*算法是网格搜索）
2. 本发明针对"掉头脱困场景"（A\*算法是通用路径规划）

**技术效果**:

1. 运动基元采样能够更好地适应狭小空间
2. 专门针对掉头场景，轨迹更合理

**实际解决的技术问题**:
"如何在掉头路段拥堵、可行区域狭小的场景下，生成合理的脱困轨迹，提高脱困成功率"

## 第三步：判断是否显而易见（技术启示）

### 3.1 技术启示判断的五个方面

#### 方面1: 在现有技术中寻找技术手段

**基本规则**: 围绕发明**实际解决的技术问题**有目的的寻找技术启示

**判断要点**:

1. **技术手段的作用必须与发明中一致**
   - 形式相似不等于技术启示
   - 作用不同 → 不构成技术启示

2. **避免碎片化拼凑**
   - 从多篇现有技术中分别抽离零散技术特征
   - 简单拼凑 ≠ 技术启示
   - 需要有结合以解决技术问题的启示

**典型案例**:

##### 案例1: 技术手段作用不同（第116169号复审决定）

**争议**: 对比文件2中用于电连接的通孔能否给出固定端子防断开的技术启示

**决定**:

- 对比文件2的通孔用于: 穿设螺钉实现电连接
- 涉案申请的通孔用于: 物理固定防止脱出
- **结论**: 虽然形式相似，但因解决的问题完全不同，不构成技术启示

**启示**: 技术手段在不同方案中作用不同时，不构成技术启示

##### 案例2: 碎片化拼凑（第28914号无效决定）

**争议**: 从六篇对比文件中分别抽离的零散技术特征能否拼凑得到发明的技术手段

**决定**:

- 各对比文件仅分别公开了保险丝、不同长度导线等单个特征
- 没有给出为解决"防止短路/反接"而将这些特征结合的启示
- **结论**: 将散落于多篇现有技术中的零散技术特征简单拼凑，明显有悖于创造性整体考虑的审查思路

**启示**: 技术特征碎片化拼凑不等于技术启示

#### 方面2: 改进动机的考量

**基本公式**:

```
改进动机 = 普遍需求 + 公知手段 + 合理成功预期
```

**产生改进动机的条件**:

1. **面对公认的问题或普遍需求**
2. **技术人员知道现有技术中存在相关技术手段**
3. **能合理预期应用成功**

**判断要点**:

1. **最接近现有技术教导改进方向** → 改进动机更强
2. **基于申请日以前的现有技术** → 避免事后诸葛亮
3. **现有技术长期未被改进的事实** → 可佐证缺乏改进动机

**典型案例**:

##### 案例1: 普遍需求产生动机（第12964号无效决定）

**争议**: 将热管和基板设为相同平面度是否显而易见

**分析**:

- 提高热传导效率是热管领域**普遍需求**
- 公知接触面平坦程度相同可**增大接触面积**
- 对比文件1和2都公开了热管和热源接触面是平的
- 技术人员很容易想到设为相同平面度

**结论**: 普遍需求 + 公知手段 + 合理成功预期 = 改进动机 ✅

##### 案例2: 避免事后诸葛亮（第27258号无效决定）

**争议**: 将5~8元环扩展到9~15元环是否有动机

**分析**:

- 对比文件1和2均将稠合环限定在5~8元
- 该范围环结构稳定，而9~15元大环易发生构象变化
- 两类环结构差异**阻碍替换**
- 对比文件公开多年间**无人尝试过扩展**

**结论**: 现有技术长期未被改进的事实可佐证缺乏改进动机

#### 方面3: 发明构思的影响

**核心原则**: 把握发明构思对于避免技术启示判断中的主观性具有重要作用

**判断标准**:

```
发明构思不同 → 形式相似 ≠ 技术启示
```

**典型案例**:

##### 案例: 场发射背光装置（第131807号复审决定）

**对比文件**: 场发射背光装置（荧光层是必需组件，电场驱动发光）

**本发明**: 直接发光型背光模组（荧光层用于改善均匀度，光子激发）

**分析**:

- 虽然都有荧光层和两个基板（**形式相似**）
- 但发明构思**完全不同**:
  - 对比文件: 场发射发光（荧光层是必需组件）
  - 本发明: 发光器件光子激发荧光粉（荧光层用于改善均匀度）

**结论**: 不同构思下的技术启示判断不能仅看表面相似

#### 方面4: 准确把握现有技术公开内容

**基本规则**: 应立足现有技术解决的技术问题、技术手段、技术效果进行**综合分析**，不能断章取义

**判断要点**:

1. **整体把握**: 不能断章取义，不能只看局部特征
2. **识别明显错误**: 现有技术中的明显错误不改变其公开内容的实质
3. **把握负面描述**: 现有技术中的负面描述（如"不宜""不适合"）有时恰恰表明该方向存在技术障碍

**典型案例**:

##### 案例: 负面描述反证无启示

**对比文件**: "本领域技术人员通常**不宜**采用X方法，因为会导致Y问题"

**分析**: 负面描述"不宜"表明该方向存在技术障碍，反而**反证**了无技术启示

#### 方面5: 合乎逻辑的分析推理和有限的试验

**5.1 逻辑推理**

**规则**: 如果发明仅是通过合乎逻辑的分析推理就能从现有技术中得到的技术方案，则通常不具备创造性

**条件**:

- 推理链条必须**严密**
- 不能**跳跃**
- 每一步推理都有依据

**5.2 有限的试验**

**规则**: 如果发明需要的是所属领域技术人员的**常规实验能力范围内的有限试验**，则创造性可能被否定

**判断标准**:

| 试验类型                   | 是否"有限" | 创造性    |
| -------------------------- | ---------- | --------- |
| 优选范围内确定最佳数值点   | ✅ 有限    | ❌ 不具备 |
| 已知范围内的常规筛选       | ✅ 有限    | ❌ 不具备 |
| 大量创造性劳动的反复试验   | ❌ 无限    | ✅ 具备   |
| 没有现有技术指引的广泛筛选 | ❌ 无限    | ✅ 具备   |

**典型案例**:

##### 案例: 化学领域优选范围

**发明**: 在已知优选范围内（如5-10%）通过有限次常规试验确定最佳数值点

**结论**: 所属领域技术人员的常规实验能力范围内，通常**不具备创造性**

## 辅助判断因素

### 4.1 辅助因素的作用

**核心原则**: 辅助判断因素有助于**保障**评判结论的准确性，但不能**单独**作为判断创造性的依据

### 4.2 三大辅助因素

#### 因素1: 长期渴望解决的技术难题

**判断标准**:

- 本领域技术人员一直渴望解决
- 长期未能获得成功
- 本发明首次解决了该难题

**典型案例**:

##### 案例: 抗生素的发现

**背景**: 细菌感染长期是人类面临的健康难题，长期渴望有效治疗手段

**发明**: 青霉素的发现

**结论**: 解决了长期渴望解决的技术难题，**具备创造性** ✅

#### 因素2: 技术偏见

**定义**: 在某个技术领域中长期存在、普遍认为**不可行**或**不适合**的技术认识

**判断标准**:

- 该偏见在发明**申请日之前**已经存在
- 在该领域中被普遍接受
- 本发明克服了该偏见

**典型案例**:

##### 案例: 金属粉末冶金

**技术偏见**: 金属粉末压制烧结后，普遍认为**不能再进行机械加工**（会破坏结构）

**发明**: 通过特殊工艺，使烧结后的金属零件能够进行机械加工

**结论**: 克服了技术偏见，**具备创造性** ✅

#### 因素3: 预料不到的技术效果

**判断标准**:

- 技术效果超出**本领域技术人员的预期**
- 不是本领域技术人员能够**合理预期**的
- 需要实验数据证明

**判断要点**:

1. **预料不到**: 必须是"出乎意料"的，不能是"略优于"
2. **技术效果**: 必须是技术效果，不是商业效果
3. **实验数据**: 需要实验数据证明（化学、生物领域）

**典型案例**:

##### 案例: 药物化合物（第13582号无效决定）

**发明**: 立普妥晶体化合物，对比文件公开了无定形立普妥

**技术效果**: 晶体形式的溶解度仅为无定形形式的1/10

**分析**:

- 所属领域技术人员通常预期晶体形式溶解度更高
- 本发明晶体溶解度**显著低于**无定形形式
- 这是**预料不到的**技术效果

**结论**: 具备创造性 ✅

## 不同类型发明的创造性判断

### 5.1 组合发明

**定义**: 将某些技术特征组合在一起，形成新的技术方案

**判断标准**:

| 情形         | 创造性判断                             |
| ------------ | -------------------------------------- | --------- |
| **简单叠加** | 各技术特征各自发挥原有功能，无协同效应 | ❌ 不具备 |
| **功能组合** | 各技术特征组合产生新的功能             | ✅ 具备   |
| **协同效应** | 组合后的技术效果**优于**各特征效果之和 | ✅ 具备   |

**典型案例**:

##### 案例: 笔 + 灯

**组合**: 将笔和灯组合在一起

**分析**:

- 笔: 书写功能
- 灯: 照明功能
- 组合后: 可以在黑暗环境中书写

**结论**: 各自发挥原有功能，**简单叠加**，**不具备创造性** ❌

##### 案例: 手机 + 摄像头 + 通信模块

**组合**: 将摄像头和通信模块集成到手机中

**分析**:

- 产生了**新的功能**: 视频通话
- 有**协同效应**: 1+1+1 > 3

**结论**: **具备创造性** ✅

### 5.2 选择发明

**定义**: 从最接近的现有技术中**有目的**地选择个别技术特征

**判断标准**:

| 情形               | 创造性判断                               |
| ------------------ | ---------------------------------------- | --------- |
| **常规选择**       | 所属领域技术人员通过有限试验就能选择到   | ❌ 不具备 |
| **预料不到的效果** | 选择产生了预料不到的技术效果             | ✅ 具备   |
| **长期未被发现**   | 该选择长期存在于现有技术中但一直未被发现 | ✅ 具备   |

### 5.3 转用发明

**定义**: 将某一技术领域的现有技术转用到其他技术领域

**判断标准**:

| 情形               | 创造性判断                   |
| ------------------ | ---------------------------- | --------- |
| **相近领域**       | 转用是显而易见的             | ❌ 不具备 |
| **遥远领域**       | 需要克服技术障碍             | ✅ 具备   |
| **预料不到的效果** | 转用产生了预料不到的技术效果 | ✅ 具备   |

**典型案例**:

##### 案例: 医用CT扫描仪转用工业检测

**分析**:

- 医用CT扫描仪: 用于人体断层扫描
- 工业检测: 用于工业产品无损检测
- 两个领域**差异较大**
- 需要克服技术障碍（精度、环境等）

**结论**: **具备创造性** ✅

### 5.4 要素变更发明

**定义**: 对现有技术的技术特征进行变更（如形状、尺寸、比例、参数等）

**判断标准**:

| 情形               | 创造性判断                           |
| ------------------ | ------------------------------------ | --------- |
| **常规优化**       | 所属领域技术人员通过有限试验就能优化 | ❌ 不具备 |
| **预料不到的效果** | 变更产生了预料不到的技术效果         | ✅ 具备   |
| **突破常规**       | 变更突破了常规认识                   | ✅ 具备   |

## 常见错误及规避

### 错误1: 事后诸葛亮

**❌ 错误做法**: 知晓发明方案后，反向寻找现有技术中的"启示"

**✅ 正确做法**: 基于申请日以前的现有技术，客观判断是否存在技术启示

**规避方法**:

- 严格按时间顺序分析
- 现有技术长期未被改进的事实可佐证缺乏改进动机

### 错误2: 碎片化拼凑

**❌ 错误做法**: 从多篇现有技术中分别抽离零散技术特征，简单拼凑

**✅ 正确做法**: 需要有**结合以解决技术问题**的启示

**规避方法**:

- 整体考虑技术方案
- 分析是否有结合动机

### 错误3: 仅看形式相似

**❌ 错误做法**: 技术手段形式相似就认为有技术启示

**✅ 正确做法**: 技术手段的**作用**必须与发明中一致

**规避方法**:

- 分析技术手段在各自方案中的作用
- 作用不同 ≠ 技术启示

### 错误4: 忽视发明构思

**❌ 错误做法**: 仅看表面特征，忽略发明构思

**✅ 正确做法**: 把握发明构思对技术启示判断的影响

**规避方法**:

- 深入理解发明人的技术思路
- 发明构思不同 → 形式相似 ≠ 技术启示

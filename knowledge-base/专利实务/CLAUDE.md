## 编译规则

### 1. 读取规则
- 必须完整读取 raw 文件后再编译，禁止只读部分
- OCR 误差处理：根据上下文推断正确内容，常见问题：
  - 数字缺失：`条` → `第5条`
  - 乱码字符：`勹` `吻竺` `咖气` → 根据上下文替换为正确汉字
  - 缺字：`明创造` → `发明创造`，`专 法` → `专利法`

### 1.1. 崔国斌数据源读取规则（数据源 2）

- 直接读取 EPUB HTML 文件，无需 OCR 处理
- 14 章对应 14 个 HTML 文件：text00005（第1章）到 text00018（第14章）
- 必须过滤比较法内容（美国、日本、欧洲专利公约等），仅保留中国法内容
- 识别比较法关键词：美国专利法、日本专利法、欧洲专利公约、TRIPS 协议等
- 比较法段落特征：讨论外国法院判例、外国专利制度对比等

### 2. 页面拆分规则
- 每个页面聚焦一个子主题
- 单个页面控制在 3000-8000 字
- 如果某节内容超过 8000 字，按子节拆分
- 页面命名格式：`{章节名}-{子主题}.md`

### 2.1. 崔国斌页面命名规则（数据源 2）

- 格式：`{主题}-原理-{子主题}.md`
- 示例：创造性-原理-显而易见的认定.md、侵权-原理-权利要求解释.md
- 与以案说法页面（无 `原理-` 前缀）区分
- 目的：同一目录下两个来源的页面清晰可辨

### 4. 案例提取规则
- 每个案例必须包含四个要素：决定编号、案件编号（如果有）、争议焦点、决定要点
- 案例中的法律条文引用必须保留具体法条号
- 启示部分要体现对专利代理实务的指导意义
- 如果原文中案例信息不完整，标注 `[信息不完整]`

### 5. 交叉引用规则
- 使用 `[[页面名]]` 格式创建 wiki 链接
- 每个页面必须关联到：同章节的其他页面 + 其他章节的相关页面
- 概念交叉示例：
  - 创造性的"区别特征" ↔ 权利要求的"保护范围"
  - 说明书的"充分公开" ↔ 权利要求的"以说明书为依据"
  - 创造性的"技术启示" ↔ 新颖性的"单独对比"

### 6. 输出规则
- 所有文件使用 UTF-8 编码
- 文件名使用中文，不含空格和特殊字符
- 每个页面独立可读，不依赖其他页面的上下文
- 术语首次出现时给出简要解释

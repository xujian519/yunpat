# 审查-计算机-说明书与权利要求撰写

> **来源：** 《专利审查指南》第二部分第九章 §5-6.3
> **对应法条：** 《专利法》第二十六条第三款、第四款
> **最新修订：** 2023年修订（新增AI模型构建/训练的说明书撰写要求、新增例20-21）
> **关联页面：** [[审查-计算机-审查基准]]、[[审查-计算机-创造性审查与审查示例]]、[[权利要求-清楚的要求]]、[[说明-原理-技术方案概括]]、[[客体-原理-计算程序]]

## 核心要点

涉及计算机程序的发明专利申请的说明书和权利要求书撰写有特殊要求。说明书必须清楚、完整地描述计算机程序的设计构思及其技术特征，应以流程图为基础按时间顺序描述各步骤。权利要求可以写成方法权利要求或产品权利要求（装置、计算机可读存储介质、计算机程序产品）。2023年修订新增了AI模型构建/训练的说明书撰写要求，强调算法与具体技术领域数据的关联描述。

## 审查标准

### 1. 说明书的撰写（§5.1 + §6.3.1）

#### 1.1 一般要求

涉及计算机程序的发明专利申请的说明书除了应当从整体上描述该发明的技术方案之外，还必须清楚、完整地描述该计算机程序的设计构思及其技术特征以及达到其技术效果的实施方式。

**流程图要求**：为了清楚、完整地描述该计算机程序的主要技术特征，说明书附图中应当给出该计算机程序的主要流程图。说明书中应当以所给出的计算机程序流程为基础，按照该流程的时间顺序，以自然语言对该计算机程序的各步骤进行描述。

**公开程度**：说明书对该计算机程序主要技术特征的描述程度应当以本领域的技术人员能够根据说明书所记载的流程图及其说明编制出能够达到所述技术效果的计算机程序为准。

**源程序**：如有必要，申请人可以用惯用的标记性程序语言简短摘录某些关键部分的计算机源程序以供参考，但不需要提交全部计算机源程序。

**硬件结构描述**：涉及计算机程序的发明专利申请包含对计算机装置硬件结构作出改变的发明内容的，说明书附图应当给出该计算机装置的硬件实体结构图，说明书应当根据该硬件实体结构图，清楚、完整地描述该计算机装置的各硬件组成部分及其相互关系。

#### 1.2 包含算法特征或商业规则和方法特征的说明书要求（§6.3.1）

包含算法特征或商业规则和方法特征的发明专利申请的说明书应当清楚、完整地描述发明为解决其技术问题所采用的解决方案。

**算法特征的撰写要求**：
- 说明书中应当写明技术特征和与其功能上彼此相互支持、存在相互作用关系的算法特征如何共同作用并且产生有益效果。
- 包含算法特征时，应当将抽象的算法与具体的技术领域结合，至少一个输入参数及其相关输出结果的定义应当与技术领域中的具体数据对应关联起来。

**商业规则和方法特征的撰写要求**：
- 包含商业规则和方法特征时，应当对解决技术问题的整个过程进行详细描述和说明，使得所属技术领域的技术人员按照说明书记载的内容，能够实现该发明的解决方案。

**有益效果的撰写**：
- 说明书应当清楚、客观地写明发明与现有技术相比所具有的有益效果，例如质量、精度或效率的提高，系统内部性能的改善等。
- 如果从用户的角度而言客观上提升了用户体验，也可以在说明书中进行说明，但应当同时说明这种用户体验的提升是如何由技术特征以及与其相互支持的算法特征或商业规则和方法特征共同带来的。

### 2. 权利要求书的撰写（§5.2 + §6.3.2）

#### 2.1 权利要求的类型

涉及计算机程序的发明专利申请的权利要求可以写成一种方法权利要求，也可以写成一种产品权利要求，例如实现该方法的装置、计算机可读存储介质或者计算机程序产品。

**无论写成哪种形式的权利要求，都必须**：
- 得到说明书的支持；
- 从整体上反映该发明的技术方案；
- 记载解决技术问题的必要技术特征；
- 不能只概括地描述该计算机程序所具有的功能和该功能所能够达到的效果。

#### 2.2 方法权利要求的撰写

如果写成方法权利要求，应当按照方法流程的步骤详细描述该计算机程序所执行的各项功能以及如何完成这些功能。

#### 2.3 装置权利要求的撰写

如果写成装置权利要求，应当具体描述该装置的各个组成部分及其各组成部分之间的关系，所述组成部分不仅可以包括硬件，还可以包括程序。

**程序模块构架的解释**：如果全部以计算机程序流程为依据，按照与该计算机程序流程的各步骤完全对应一致的方式撰写装置权利要求，则这种装置权利要求中的各组成部分应当理解为实现该程序流程各步骤所必须建立的程序模块，由这样一组程序模块限定的装置权利要求应当理解为主要通过说明书记载的计算机程序实现该解决方案的程序模块构架，而不应当理解为主要通过硬件方式实现该解决方案的实体装置。

**计算机程序产品的理解**：计算机程序产品应当理解为主要通过计算机程序实现其解决方案的软件产品。

#### 2.4 包含算法特征或商业规则和方法特征的权利要求要求（§6.3.2）

包含算法特征或商业规则和方法特征的发明专利申请的权利要求应当以说明书为依据，清楚、简要地限定要求专利保护的范围。权利要求应当记载技术特征以及与技术特征功能上彼此相互支持、存在相互作用关系的算法特征或商业规则和方法特征。

### 3. 撰写示例

#### 3.1 方法与装置权利要求的对应撰写（§5.2例1-2）

**【例1】方法权利要求——CRT屏幕游标控制方法**

一种CRT显示屏幕的游标控制方法，包括：
- 用于输入信息的输入步骤；
- 用于将游标水平和垂直移动起始位置地址存储到H/V起始位置存储装置中的步骤；
- 用于将游标水平和垂直移动终点位置地址存储到H/V终点位置存储装置中的步骤；
- 用于将游标当前位置的水平和垂直地址存储到游标位置存储装置中的步骤；
- 其特征是还包括：比较步骤、游标位置变换步骤、游标显示步骤。

**【例2】装置权利要求——CRT屏幕游标控制器**

一种CRT显示屏幕的游标控制器，包括：输入装置、H/V起始位置存储装置、H/V终点位置存储装置、游标位置存储装置等各功能模块，与方法步骤完全对应。

#### 3.2 四种权利要求类型的撰写（§5.2例4）

对于"去除图像噪声的方法"发明专利申请，可以按下述方式撰写四种类型的权利要求：

1. **方法权利要求**：一种去除图像噪声的方法，其特征在于包括获取像素数据、计算灰度均值及方差值、判断并去除噪声等步骤。

2. **装置权利要求**：一种计算机装置/设备/系统，包括存储器、处理器及存储在存储器上的计算机程序，所述处理器执行所述计算机程序以实现方法权利要求所述方法的步骤。

3. **计算机可读存储介质**：一种计算机可读存储介质，其上存储有计算机程序/指令，该计算机程序/指令被处理器执行时实现方法权利要求所述方法的步骤。

4. **计算机程序产品**：一种计算机程序产品，包括计算机程序/指令，该计算机程序/指令被处理器执行时实现方法权利要求所述方法的步骤。

## 实务应对

**代理人视角：**

1. **说明书的流程图撰写**：涉及计算机程序的发明必须在说明书附图中给出主要流程图，并按时间顺序用自然语言描述各步骤。这是计算机程序发明说明书的基本要求，缺少流程图可能被认为公开不充分。

2. **算法与数据的关联描述**：2023年修订特别强调，包含算法特征的说明书中，"至少一个输入参数及其相关输出结果的定义应当与技术领域中的具体数据对应关联起来"。撰写时应避免使用纯抽象的通用数据（如"特征值""标签值"），而应使用具体的技术术语（如"图像像素灰度值""传感器温度数据"等）。

3. **四种权利要求类型的布局**：对于重要的计算机程序发明，建议同时布局方法、装置（功能模块构架）、计算机可读存储介质和计算机程序产品四种类型的权利要求，以获得更全面的保护。

4. **装置权利要求的撰写注意**：如果采用"功能模块+程序流程步骤"的方式撰写装置权利要求，应确保各模块与流程步骤的完全对应，使权利要求被理解为"程序模块构架"而非"硬件实体装置"。

5. **有益效果的分层描述**：
   - 技术效果层面：质量、精度、效率的提高，系统内部性能的改善
   - 用户体验层面：客观的用户体验提升（需与技术特征关联）
   - 避免仅描述商业效果

6. **源程序的使用**：不需要提交全部源程序，但可以简短摘录关键部分作为参考。如果核心算法较为复杂，建议在说明书中以伪代码形式呈现关键逻辑。

### 撰写原则

1. **以说明书为依据**：权利要求应当得到说明书的支持
2. **清楚**：每项权利要求应当清楚、简要
3. **简要**：权利要求应当简要
4. **保护范围合理**：既不能过宽（缺乏新颖性/创造性），也不能过窄（容易被规避）

## 输出要求

请按以下格式输出权利要求书：

### 1. 独立权利要求（1个）

**格式**：

```
1. 一种[技术主题]，其特征在于，包括：
[特征A]；
[特征B]；
[特征C]。
```

**要求**：

- 包含发明的所有必要技术特征
- 特征之间用分号分隔
- 结尾用句号
- 保护范围合理

### 2. 从属权利要求（{{claims_count}}个）

**格式**：

```
2. 根据权利要求1所述的[技术主题]，其特征在于：
[附加特征A]；
[附加特征B]。
```

**要求**：

- 引用所依附的权利要求
- 包含附加技术特征
- 进一步限定保护范围
- 层次清晰

## 质量检查清单

撰写完成后，请检查：

- [ ] 独立权利要求包含所有必要特征
- [ ] 从属权利要求正确引用独立权利要求
- [ ] 每项权利要求清楚、简要
- [ ] 权利要求得到说明书支持
- [ ] 保护范围既不过宽也不过窄
- [ ] 特征描述具体，避免模糊表达
- [ ] 术语统一，前后一致

## 🔴 高优先级 Agent（强烈推荐集成）

### 1️⃣ prior-art-search - 先导技术检索

**功能**：构建检索策略并分析现有技术

**为什么需要知识图谱**：

- ✅ **法律条文检索** - 检索专利法、审查指南中关于新颖性、创造性的规定
- ✅ **无效决定案例** - 查询类似的无效宣告决定（9,562 条）
- ✅ **判例法理** - 查询相关判决书（5,906 条）
- ✅ **核心概念** - YunPat 100 个核心概念（如"现有技术"、"等同侵权"）

**知识增强场景**：

```typescript
// 查询时自动检索相关法律知识
const knowledge = await this.queryKnowledge('新颖性判断标准', 10)
// 返回：
// - 专利法第22条（新颖性规定）
// - 审查指南第二部分第三章（新颖性审查）
// - 相关无效决定案例
// - "新颖性"核心概念及定义
```

**价值提升**：

- 检索策略更准确（基于法律条文和案例）
- 现有技术分析更深入（结合无效决定判例）
- 避免遗漏关键法律条款

---

### 2️⃣ patent-responder - OA 审查意见答复

**功能**：OA 审查意见答复与策略生成

**为什么需要知识图谱**：

- ✅ **答复策略** - 基于相似案例的答复策略（9,562 条无效决定）
- ✅ **法律依据** - 专利法、审查指南相关规定
- ✅ **判例参考** - 相关判决书的说理方式
- ✅ **概念理解** - "创造性"、"显而易见"等核心概念

**知识增强场景**：

```typescript
// 审查员认为"创造性不足"
const knowledge = await this.queryKnowledge('创造性争辩 策略', 10)
// 返回：
// - 成功争辩的无效决定案例
// - 创造性判断的法律标准
// - 常用的争辩理由和技巧
// - "三步法"等核心概念
```

**价值提升**：

- 答复策略更有针对性（基于成功案例）
- 法律依据更充分（检索相关法条）
- 争辩理由更有说服力（结合判例法理）

---

### 3️⃣ claim-generator - 权利要求生成

**功能**：基于发明理解和检索分析撰写权利要求

**为什么需要知识图谱**：

- ✅ **撰写规范** - 专利法、审查指南关于权利要求的规定
- ✅ **无效决定参考** - 避免常见的无效宣告理由（9,562 条）
- ✅ **权利要求解释** - 相关判决书对权利要求的解释
- ✅ **核心概念** - "独立权利要求"、"从属权利要求"等

**知识增强场景**：

```typescript
// 撰写权利要求时
const knowledge = await this.queryKnowledge('权利要求撰写规范', 10)
// 返回：
// - 专利法第26条第4款（权利要求规定）
// - 审查指南第二部分第二章（权利要求审查）
// - 常见无效理由：不清楚、不支持、缺必要技术特征
// - 权利要求示例和模板
```

**价值提升**：

- 权利要求更规范（符合法律要求）
- 避免常见无效理由（基于无效决定案例）
- 保护范围更合理（结合审查指南）

---

### 4️⃣ specification-drafter - 说明书撰写

**功能**：分章节撰写专利说明书

**为什么需要知识图谱**：

- ✅ **撰写要求** - 专利法、审查指南关于说明书的规定
- ✅ **实施例参考** - 无效决定中实施例的描述方式
- ✅ **技术术语** - 相关判决书中的技术术语解释
- ✅ **核心概念** - "充分公开"、"实施例"等

**知识增强场景**：

```typescript
// 撰写实施例时
const knowledge = await this.queryKnowledge('实施例撰写要求', 10)
// 返回：
// - 专利法第26条第3款（充分公开要求）
// - 审查指南第二部分第五章（实施例审查）
// - 无效决定中实施例不足的案例
// - 实施例撰写模板和技巧
```

**价值提升**：

- 说明书更完整（符合充分公开要求）
- 实施例更充分（避免支持不足）
- 技术描述更准确（参考标准表述）

---

### 5️⃣ patent-invalidity - 专利无效宣告分析（如果存在）

**功能**：专利无效宣告策略与分析

- ✅ **成功案例** - 成功无效宣告的策略和理由
- ✅ **核心概念** - "无效理由"、"证据组合"等

```typescript
// - 证据组合策略
```

**价值提升**：

- 无效理由更充分（基于成功案例）
- 策略更精准（避免常见错误）
- 成功率更高（参考最优实践）

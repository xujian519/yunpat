## 核心问题

### 当前痛点

1. **硬编码问题**：提示词在 Agent 类中，修改需重编译（10分钟/次）
2. **知识库脱节**：134+ 知识卡片、500+ Wiki 页面利用率仅 30%
3. **缺乏元数据**：无使用时机、工具限制、条件激活
4. **无动态能力**：无法运行时发现、加载、激活

### 影响

- 开发效率低：提示词迭代慢
- 知识库浪费：大量知识未利用
- 用户体验差：需手动选择技能
- API 成本高：无缓存优化

---

## 优化方案

### 核心设计：Skills 系统

```
SKILL.md (Markdown + Frontmatter)
    ↓
SkillLoader (加载器)
    ↓
KnowledgeBridge (知识库桥接)
    ↓
ConditionalActivator (条件激活)
    ↓
Agent (调用)
```

### 关键特性

1. **Markdown + Frontmatter**
   - 提示词与代码分离
   - 丰富的元数据支持
   - 修改无需重编译

2. **知识库深度集成**
   - 概念查询：`{{#knowledge.concepts}}`
   - Wiki 页面：`{{#knowledge.wiki_pages}}`
   - 知识卡片：`{{#knowledge.cards}}`

3. **条件自动激活**
   - 基于文件路径：`paths: ["**/*.pdf"]`
   - 基于文件类型：`file-types: ["pdf"]`
   - 自动匹配，自动激活

4. **Shell 命令注入**
   - 动态内容：`!`git status``
   - 变量嵌套：`!`cat {{file_path}}``
   - 安全白名单

5. **分层缓存**
   - 静态内容：全局缓存
   - 动态内容：会话特定
   - 降低 API 成本 30%

## 快速开始

### 5 分钟创建第一个 Skill

```bash
# 1. 创建目录
mkdir -p .yunpat/skills/hello-world

# 2. 编写 SKILL.md
cat > .yunpat/skills/hello-world/SKILL.md << 'EOF'
---
name: hello-world
description: 向用户打招呼
---

你是一位友好的专利助手。请用温暖的语气向用户打招呼。
EOF

# 3. 在 Agent 中调用
await this.callSkill('hello-world', {})
```

**恭喜！** 你已经创建了第一个 Skill 🎉

# 审查-无效-无效程序中的修改

> **来源：** 《专利审查指南》第四部分第三章 §4.6
> **对应法条：** 《专利法》第33条 / 《专利法实施细则》第69条、第73条
> **最新修订：** 2023年修订（新增§4.6.4修改文本提交规则）
> **关联页面：** [[审查-无效-审查范围与理由变更]]、[[程序-原理-无效程序中的修改]]、[[程序-无效宣告请求的审查]]

---

## 核心要点

无效宣告程序中，发明或实用新型专利文件的修改仅限于权利要求书，且应当针对无效宣告理由或合议组指出的缺陷进行修改。修改方式一般限于权利要求的删除、技术方案的删除、权利要求的进一步限定和明显错误的修正。2023年修订新增了修改文本提交的具体规则。

---

## 审查标准

### 一、修改原则（§4.6.1）

发明或者实用新型专利文件的修改仅限于权利要求书，且应当针对无效宣告理由或者合议组指出的缺陷进行修改，其原则是：

1. **不得改变原权利要求的主题名称。**
2. **与授权的权利要求相比，不得扩大原专利的保护范围。**
3. **不得超出原说明书和权利要求书记载的范围。**
4. **一般不得增加未包含在授权的权利要求书中的技术特征。**

外观设计专利的专利权人不得修改其专利文件。

#### 1.1 修改原则的详细解读

| 原则 | 核心含义 | 常见违规情形 |
|------|---------|-------------|
| 不改变主题名称 | 修改后的权利要求主题名称应与授权时保持一致 | 将"一种装置"改为"一种方法"；将"制备方法"改为"用途" |
| 不扩大保护范围 | 修改后的权利要求的保护范围不得大于授权时的保护范围 | 删除限制性技术特征；将具体特征上位化 |
| 不超原说明书范围 | 修改后的内容应在原说明书和权利要求书中有明确记载或可直接确定 | 新增原申请文件中未记载的技术特征；改变数值范围 |
| 不增加授权未含特征 | 一般不得将仅在说明书中记载但未在授权权利要求中出现的技术特征补入权利要求 | 将从属权利要求中的特征补入独立权利要求属于"进一步限定"，通常允许；但增加全新的技术特征一般不允许 |

### 二、修改方式（§4.6.2）

在满足上述修改原则的前提下，修改权利要求书的具体方式一般限于：

1. **权利要求的删除**：从权利要求书中去掉某项或者某些项权利要求，例如独立权利要求或者从属权利要求。
2. **技术方案的删除**：从同一权利要求中并列的两种以上技术方案中删除一种或者一种以上技术方案。
3. **权利要求的进一步限定**：在权利要求中补入其他权利要求中记载的一个或者多个技术特征，以缩小保护范围。
4. **明显错误的修正**。

#### 2.1 各种修改方式的详细说明

**权利要求的删除**：

权利要求的删除是最直接的修改方式，可以是删除一项或多项独立权利要求，也可以是删除一项或多项从属权利要求。删除后，剩余的权利要求之间仍需满足单一性要求（若涉及多项权利要求）。

**技术方案的删除**：

适用于一项权利要求中并列限定多个技术方案的情形（如"A或B"式的并列选择）。专利权人可以删除其中部分技术方案，保留其余技术方案。例如，权利要求限定"材料为X、Y或Z"，专利权人可以删除"Z"，仅保留"X或Y"。

**权利要求的进一步限定**：

"权利要求的进一步限定"是指在权利要求中补入其他权利要求中记载的一个或者多个技术特征，以缩小保护范围。这是无效程序中最常用的修改方式。

例如，独立权利要求1包含特征A+B，从属权利要求2进一步限定特征C，从属权利要求3进一步限定特征D。专利权人可以将特征C和/或D补入权利要求1中，形成新的独立权利要求1'（A+B+C或A+B+D或A+B+C+D）。

**明显错误的修正**：

明显错误是指授权权利要求书中存在的、本领域技术人员能够立即识别并确定正确内容的错误。例如：
- 错别字或标点符号错误；
- 附图标记的错误引用；
- 数值的明显笔误（如将"100℃"误写为"1000℃"，且从上下文可确认应为100℃）。

### 三、修改方式的限制（§4.6.3）

**时间限制**：

在审查决定作出之前，专利权人可以删除权利要求或者权利要求中包括的技术方案。

仅在下列三种情形的**答复期限内**，专利权人可以以删除以外的方式修改权利要求书：

1. **针对无效宣告请求书**。
2. **针对请求人增加的无效宣告理由或者补充的证据**。
3. **针对合议组引入的请求人未提及的无效宣告理由或者证据**。

#### 3.1 修改时间限制的实务理解

| 修改方式 | 时间限制 | 说明 |
|---------|---------|------|
| 权利要求删除 | 审查决定作出前均可 | 灵活性最大，可在程序任何阶段进行 |
| 技术方案删除 | 审查决定作出前均可 | 同权利要求删除 |
| 进一步限定 | 仅在上述三种答复期限内 | 需把握时机，逾期不可再以该方式修改 |
| 明显错误修正 | 一般应在答复期限内 | 若错误明显且不影响审理，审查员可能允许在决定前随时修正 |

### 四、修改文本的提交（§4.6.4）[2023年新增]

> **[2023年新增条款]**

专利权人提交修改文本的，应当以**替换页**的方式提交，并同时提交**修改对照页**。

专利权人在无效宣告程序中多次提交修改文本的，**以最后一次提交的为准**，但前述修改方式不受限制的权利要求删除和技术方案删除除外。

#### 4.1 修改文本提交规则详解

**替换页**：指提交完整的修改后的权利要求书全文，用以替换之前的权利要求书文本。

**修改对照页**：指清晰显示修改前后对照的页面，便于审查员和请求人快速了解修改内容。通常包括：
- 修改前的权利要求原文（可标记删除线）；
- 修改后的权利要求文本（可标记下划线或加粗）；
- 简要说明修改的理由和依据。

**"以最后一次为准"规则的理解**：

| 情形 | 处理规则 |
|------|---------|
| 多次提交进一步限定的修改文本 | 以最后一次提交的文本为准，之前的修改文本视为未提交 |
| 多次提交权利要求删除或技术方案删除 | 不受"以最后一次为准"限制，各次删除均有效（只要删除的是不同权利要求） |
| 最后一次提交的修改文本不符合规定 | 合议组可不予接受，并可要求专利权人在指定期限内重新提交 |

### 五、修改超范围的判断标准

无效程序中的修改超出原说明书和权利要求书记载的范围，是常见的修改缺陷。判断修改是否超范围的核心标准：

#### 5.1 判断是否超范围的方法

修改后的内容是否符合专利法第33条，应当以原说明书和权利要求书记载的内容为准。具体判断方法：

1. **直接记载标准**：修改后的技术特征应当能够在原说明书和权利要求书中找到完全相同的文字记载；
2. **直接地、毫无疑义地确定标准**：即使未找到完全相同的文字记载，如果所属技术领域的技术人员能够从原说明书和权利要求书中直接地、毫无疑义地确定该技术特征，则修改不超范围；
3. **禁止引入新内容**：不得通过修改引入原申请文件中未记载的技术内容，即使该内容属于本领域公知常识。

#### 5.2 常见超范围情形

| 修改类型 | 是否可能超范围 | 说明 |
|---------|--------------|------|
| 将从属权利要求的特征补入独立权利要求 | 一般不超范围 | 特征已在授权权利要求书中记载 |
| 将说明书中的特征补入权利要求 | 可能超范围 | 若该特征未出现在授权权利要求书中，一般不允许 |
| 改变数值范围 | 可能超范围 | 若新范围的端点在原文件中未明确记载，可能超范围 |
| 增加功能性限定 | 可能超范围 | 若功能未被原权利要求限定，可能属于增加新特征 |
| 修正明显错误 | 不超范围 | 错误内容和正确内容均可从上下文唯一确定 |

### 六、不同类型权利要求的修改规则

#### 6.1 产品权利要求的修改

- 可以通过增加结构特征、材料特征或连接关系特征来缩小保护范围；
- 不得将产品权利要求改为方法权利要求（改变主题名称）；
- 不得删除必要技术特征导致保护范围扩大。

#### 6.2 方法权利要求的修改

- 可以通过增加工艺步骤、工艺条件或设备特征来缩小保护范围；
- 不得将方法权利要求改为产品权利要求（改变主题名称）；
- 删除方法步骤时应注意是否扩大了保护范围（删除步骤通常会使方法更宽泛，可能扩大保护范围，需谨慎）。

#### 6.3 用途权利要求的修改

- 用途权利要求的修改应特别注意主题名称不得改变；
- 增加用途限制或适应症限制通常属于缩小保护范围，一般允许；
- 将"用途"改为"方法"或反之，属于改变主题名称，不允许。

### 七、审查示例

**【示例1】权利要求的进一步限定**

授权权利要求：
- 权利要求1：一种化合物X，用于治疗疾病Y。
- 权利要求2：如权利要求1所述的化合物X，其中给药剂量为每日10-50mg。

无效请求理由：权利要求1不具备创造性。

专利权人修改：将权利要求2的特征补入权利要求1，修改为"一种化合物X，用于治疗疾病Y，给药剂量为每日10-50mg"。

审查结论：该修改属于"权利要求的进一步限定"，缩小了保护范围，符合修改原则。

**【示例2】技术方案的删除**

授权权利要求1：一种涂料，包含A或B或C作为颜料。

无效请求理由：对比文件公开了包含A的涂料，权利要求1中限定A的技术方案不具备新颖性。

专利权人修改：删除权利要求1中"包含A"的技术方案，保留"包含B或C"的技术方案。

审查结论：该修改属于"技术方案的删除"，符合修改方式的规定。

**【示例3】修改超范围**

授权权利要求1：一种设备，包括部件A和B。

专利权人修改：在权利要求1中增加特征C，但特征C仅在说明书中提及，未在授权的任何权利要求中出现。

审查结论：该修改属于"增加未包含在授权权利要求书中的技术特征"，不符合§4.6.1(4)的原则，不予接受。

## 实务应对

**代理人视角：**

1. **修改时机的把握**：以删除以外的方式修改权利要求书只能在三种特定的答复期限内进行，专利权人应充分利用这些时机。尤其是针对无效宣告请求书的答复期限，是最重要的修改窗口。建议专利权人在收到无效宣告请求书后，立即评估修改策略，并在首次答复期内提交修改文本。

2. **修改策略的选择**：
   - **删除方式**（权利要求删除、技术方案删除）在审查决定作出前均可进行，灵活性最大。
   - **进一步限定**只能在上述三种答复期限内进行，需要把握时机。
   - 建议先以进一步限定方式缩小保护范围，若仍无法克服缺陷再考虑删除。
   - 注意：删除独立权利要求后，需确保剩余的从属权利要求之间仍具有单一性。

3. **修改文本的提交规范**（2023年新增要求）：
   - 必须以替换页方式提交修改文本，并同时提交修改对照页。
   - 多次提交时以最后一次为准，因此应在每次提交时确保修改文本的完整性和准确性。
   - 注意：权利要求删除和技术方案删除不受"以最后一次为准"的限制。
   - 修改对照页应清晰标示修改前后内容，便于审查员和对方当事人理解。

4. **外观设计专利的特殊性**：外观设计专利权人在无效程序中不得修改专利文件，这与发明和实用新型专利形成鲜明对比。外观设计专利权人只能选择放弃权利要求或坚持原有设计。

5. **"进一步限定"的运用**：通过补入从属权利要求中的技术特征来缩小独立权利要求的保护范围，是最常用的修改策略。但应注意补入的技术特征必须来源于其他权利要求中已记载的内容，避免引入超范围的新特征。

6. **避免修改超范围的注意事项**：
   - 修改前仔细核对拟补入的特征是否已在授权权利要求书中出现；
   - 不要试图通过修改引入仅在说明书中记载而未在权利要求中出现的特征；
   - 数值范围的修改应确保新范围的端点在原文件中有依据；
   - 修改后的权利要求之间应保持单一性。

## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 修改改变了权利要求主题名称 | 不符合修改原则 | §4.6.1(1) |
| 修改扩大了授权专利的保护范围 | 不符合修改原则 | §4.6.1(2) |
| 在非指定答复期限以删除以外方式修改 | 不予接受 | §4.6.3 |
| 外观设计专利权人提交修改文本 | 不允许（外观设计不得修改） | §4.6.1 |
| 多次提交修改文本 | 以最后一次为准（删除除外） | §4.6.4 |
| 未提交修改对照页 | 应当补充提交 | §4.6.4 |
| 修改增加了授权权利要求书中未包含的技术特征 | 一般不符合修改原则 | §4.6.1(4) |
| 权利要求的进一步限定（补入从属权利要求特征） | 一般允许 | §4.6.2(3) |
| 技术方案的删除 | 审查决定作出前均可 | §4.6.3 |
| 明显错误的修正 | 允许 | §4.6.2(4) |
| 修改超出原说明书和权利要求书记载范围 | 不符合修改原则 | §4.6.1(3) |

## 核心撰写原则（基于知识库）

### 1. 清楚性原则

**法律依据**: 《专利法实施细则》第20条

**核心要求**:

- 权利要求的类型应当清楚（产品 vs 方法）
- 权利要求中的用词应当清楚
- 使用符合国家统一规定的技术用语
- 不允许使用土语、自行编造的词语
- 不得使用含糊不清的用语，如"厚""薄""强""弱""高温""高压"等（除非在固定领域中具有公认的确切含义）
- **不得出现**: "例如""最好是""尤其是""必要时"等对保护范围不起限定作用的用语
- **谨慎使用**: "约""接近"等导致保护范围不清楚的用词

**产品权利要求 vs 方法权利要求**:

| 类型             | 定义                             | 保护特点                 | 撰写模板                                                                 |
| ---------------- | -------------------------------- | ------------------------ | ------------------------------------------------------------------------ |
| **产品权利要求** | 人类通过技术手段生产出的任何物体 | 保护是绝对的，易核查     | "一种[装置名称]，其特征在于：包括[组件1]；与所述[组件1]连接的[组件2]..." |
| **方法权利要求** | 人类通过技术手段进行的一系列活动 | 保护相对较弱，需现场取证 | "一种[方法名称]，其特征在于，包括以下步骤：[步骤1]；[步骤2]..."          |

### 2. 简要性原则

**核心要求**:

- 用词应当简明扼要
- 除记载技术特征外，不得对原因或理由作不必要的描述
- 权利要求的数目应当合理，避免不必要的重复

### 3. 支持性原则（A26.4）

**法律依据**: 《专利法》第26条第4款

**核心要求**:

- 权利要求书应当以说明书为依据
- 权利要求的概括不得超出说明书充分公开的内容范围
- 使本领域技术人员能够合理预期其技术效果
- **判断标准**: 所属技术领域的技术人员能够从说明书充分公开的内容中得到或概括得到的技术方案，并且不超出说明书公开的范围

### 4. 必要技术特征原则

**法律依据**: 《专利法实施细则》第21条第2款

**定义**: 必要技术特征是指发明或者实用新型为解决其技术问题所**不可缺少**的技术特征

**三个关键要素**:
| 要素 | 内容 |
|------|------|
| **不可或缺性** | 缺少该技术特征，就无法解决发明提出的技术问题 |
| **整体性** | 全部必要技术特征的总和构成完整的技术方案 |
| **区别性** | 必要技术特征的总和使发明区别于其他技术方案 |

**确定方法**（从技术问题出发）:

1. 明确发明要解决的技术问题
2. 分析说明书中描述的技术方案
3. 逐一判断每个技术特征是否为解决该技术问题所必需
4. 将必要技术特征写入独立权利要求

## 两部分撰写法

### 标准结构

```
独立权利要求 = 前序部分 + 特征部分

前序部分 = 发明名称 + 与现有技术共有的必要技术特征
特征部分 = "其特征是……" / "其特征在于" + 区别于现有技术的技术特征
```

### 撰写步骤

#### Step 1: 确定发明原型（最接近的现有技术）

用作"划界"的最相关的一件现有技术文件是客观存在的，不是以人们的主观意志来决定的。

#### Step 2: 撰写前序部分

- 写明发明或者实用新型要求保护的主题名称
- 写明发明或者实用新型主题与现有技术共有的必要技术特征

#### Step 3: 撰写特征部分

- 使用"其特征是……"或"其特征在于……"或类似的用语
- 写明发明或者实用新型区别于现有技术的技术特征
- 这些特征和前序部分写明的特征合在一起，限定发明或者实用新型要求保护的范围

### "划界"与创造性的关系

- **全新的发明**: 不存在最接近它的现有技术，如果按两部分形式撰写，所有技术特征都写在特征部分，那么具备创造性的可能性就大
- **所有特征都被划到前序部分**: 如果用最接近的一件现有技术进行划界，所有技术特征都被划到前序部分，那么该项发明的技术方案连新颖性都不具备，也就谈不上创造性了

**注意**: 也不是特征部分的技术特征少就一定不具备创造性，有时特征部分只有一个技术特征，也可能是一件创造性极高的发明。

## 不同类型发明的撰写模板

### 1. 装置/设备类发明

**模板**:

```
一种[装置名称]，其特征在于：包括[组件A]；与所述[组件A]连接的[组件B]；设置在所述[组件B]上的[组件C]；以及[特征D]，所述[特征D]用于[功能描述]。
```

**示例**:

```
一种便携式牙刷，其特征在于：包括牙刷本体；与所述牙刷本体连接的盒体；设置在所述盒体内的牙膏软袋；以及挤压装置，所述挤压装置用于挤压所述牙膏软袋；其中，所述牙膏软袋的出膏口位置与所述牙刷本体的刷毛空腔位置相对应。
```

### 2. 方法类发明

**模板**:

```
一种[方法名称]，其特征在于，包括以下步骤：
[步骤1]；
[步骤2]，所述[步骤2]包括[具体操作]；
[步骤3]；
其中，[条件或限制]。
```

**示例**:

```
一种自动驾驶掉头脱困规划方法，其特征在于，包括以下步骤：
检测车辆是否处于受困场景；
若处于受困场景，则识别障碍物分布信息；
基于所述障碍物分布信息，采样运动基元，所述运动基元包括车辆位置、航向角和转弯半径；
根据所述运动基元，生成掉头脱困轨迹；
执行所述掉头脱困轨迹。
```

### 3. 系统类发明

**模板**:

```
一种[系统名称]，包括：
[模块1]，被配置为[功能1]；
[模块2]，被配置为[功能2]；与所述[模块1]通信连接的[模块3]，被配置为[功能3]；
其中，[模块1]还被配置为[附加功能]。
```

### 4. 组合物类发明

**模板**:

```
一种[组合物名称]，其特征在于，包括：
[组分A]，其含量为[范围1]；
[组分B]，其含量为[范围2]；
以及[组分C]，其含量为[范围3]。
```

## 质量检查清单

生成权利要求后，请逐一检查：

### ✅ 清楚性检查

- [ ] 权利要求类型是否清楚（产品/方法）？
- [ ] 所有技术术语是否准确？
- [ ] 是否避免了"例如""最好是"等不起限定作用的用语？
- [ ] 是否避免了"厚""薄""高温"等含糊用语（除非有公认含义）？
- [ ] 保护范围的边界是否明确？

### ✅ 支持性检查（A26.4）

- [ ] 所有技术特征是否都在说明书中公开？
- [ ] 权利要求的概括是否超出说明书公开的范围？
- [ ] 本领域技术人员能否合理预期技术效果？
- [ ] 是否存在功能性限定（如有，是否必要）？

### ✅ 必要技术特征检查

- [ ] 独立权利要求是否包含所有必要技术特征？
- [ ] 每个必要技术特征是否不可或缺？
- [ ] 缺少任一必要技术特征是否无法解决技术问题？

### ✅ 创造性预判

- [ ] 独立权利要求与最接近现有技术的区别特征是否明确？
- [ ] 区别特征是否未被公开？
- [ ] 是否存在技术启示？
- [ ] 是否产生了预料不到的技术效果？

### ✅ 撰写形式检查

- [ ] 是否使用两部分撰写法？
- [ ] 前部分是否准确反映了发明原型？
- [ ] "其特征在于"是否正确划界？
- [ ] 从属权利要求是否正确引用？

## 常见错误及规避

### 错误1: 功能性限定过度

**❌ 错误示例**:

```
一种固定装置，其特征在于：包括固定单元，用于固定物体。
```

**✅ 正确示例**:

```
一种固定装置，其特征在于：包括夹持部，所述夹持部包括第一夹臂和第二夹臂，所述第一夹臂和所述第二夹臂之间形成夹持空间。
```

### 错误2: 缺少必要技术特征

**❌ 错误示例**: 便携式牙刷权利要求中缺少"牙膏软袋出膏口位置与刷毛空腔位置相对应"这一关键特征

**✅ 正确做法**: 从技术问题出发，逐一判断每个技术特征是否必需

### 错误3: 概括过度导致不支持

**❌ 错误示例**: 说明书只公开了"金属"材料，权利要求概括为"固体材料"

**✅ 正确做法**: 权利要求的概括不得超出说明书充分公开的范围

### 错误4: 权利要求类型不匹配

**❌ 错误示例**: 发明主题是"提升运输装置"，却写成"一种提升运输方法"

**✅ 正确示例**: 根据发明的客观性质确定权利要求类型

## 示例案例

### 案例1: 便携式牙刷（机械类）

**技术问题**: 不取出牙膏软袋即可将牙膏挤在刷毛上，使用方便

**独立权利要求**:

```
1. 一种便携式牙刷，其特征在于：包括牙刷本体；与所述牙刷本体活动连接的盒体；设置在所述盒体内的牙膏软袋；以及挤压装置，所述挤压装置用于挤压所述牙膏软袋；其中，所述牙膏软袋设有出膏口，所述牙刷本体设有用于容纳刷毛的空腔，所述出膏口的位置与所述空腔的位置相对应。
```

**从属权利要求**:

```
2. 如权利要求1所述的便携式牙刷，其特征在于：所述盒体顶壁设有用于容纳所述刷毛的空腔。

3. 如权利要求1所述的便携式牙刷，其特征在于：所述挤压装置包括设置在所述盒体侧壁的按压部。

4. 如权利要求1所述的便携式牙刷，其特征在于：所述牙刷本体与所述盒体通过螺纹连接。
```

### 案例2: 自动驾驶掉头方法（方法类）

**技术问题**: 掉头路段拥堵、停车会车道路拥堵时，车辆局部受困

**独立权利要求**:

```
1. 一种自动驾驶掉头脱困规划方法，其特征在于，包括以下步骤：
检测车辆是否处于受困场景；
若处于受困场景，则识别障碍物分布信息；
基于所述障碍物分布信息，从预设的运动基元集合中采样运动基元，所述运动基元包括车辆位置、航向角和转弯半径；
根据所述运动基元，生成掉头脱困轨迹；以及
执行所述掉头脱困轨迹。
```

## 注意事项

1. **用语一致性**: 权利要求书中使用的科技术语应当与说明书中使用的科技术语一致
2. **附图标记**: 可以引用说明书附图中相应的标记，该标记应当放在相应的技术特征后面，并置于括号内，以利于理解权利要求。**附图标记不得解释为对权利要求的限制**
3. **非技术内容**: 权利要求书中不应当含有涉及商业利益或其它非技术内容的任何说明
4. **方法-装置对应**: 对于产品发明，可以同时撰写产品权利要求和制造该产品的方法权利要求，但不允许写成"一种产品及其方法"

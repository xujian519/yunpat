## 架构设计

### 1. 分层存储策略

**推荐做法**：

- 短期记忆：对话历史、临时数据（内存）
- 长期记忆：知识库、历史数据（PostgreSQL + pgvector）
- 检查点：关键状态、时间旅行（CheckpointManager）

**示例**：

```typescript
// 短期记忆：对话上下文
const shortTerm = new MemoryManager()
await shortTerm.add('conversation', messages)

// 长期记忆：持久化知识
const vectorStore = new PostgresVectorStore({ databaseUrl })
await vectorStore.upsert({
  type: 'knowledge',
  content: '专利知识',
  embedding: embedding,
})

// 检查点：关键状态
const checkpoint = await checkpointManager.saveCheckpoint(
  'Agent',
  'exec-1',
  10,
  memory,
  context,
  state
)
```

### 2. 单一职责原则

**推荐做法**：

- MemoryManager：只负责短期记忆
- PostgresVectorStore：只负责向量存储
- CheckpointManager：只负责检查点

**避免**：

```typescript
// ❌ 不推荐：混合职责
class SuperManager {
  async saveToMemory() { ... }
  async saveToPostgres() { ... }
  async createCheckpoint() { ... }
}
```

**推荐**：

```typescript
// ✅ 推荐：职责分离
const memory = new MemoryManager()
const vectorStore = new PostgresVectorStore({ databaseUrl })
const checkpointManager = new CheckpointManager()
```

## 性能优化

### 1. 批量操作

**推荐做法**：

```typescript
// ✅ 批量插入
const items = [
  { type: 'patent', content: '...', embedding: vec1 },
  { type: 'patent', content: '...', embedding: vec2 },
  { type: 'patent', content: '...', embedding: vec3 },
]
await vectorStore.upsertBatch(items)
```

**避免**：

```typescript
// ❌ 逐条插入（慢）
for (const item of items) {
  await vectorStore.upsert(item)
}
```

### 2. 缓存策略

**推荐做法**：

```typescript
// BGE-M3 内置 LRU 缓存
const bge = createBGEM3Client({
  cacheMaxSize: 1000, // 限制缓存大小
})

// 多次调用同一文本，命中缓存
const vec1 = await bge.embed('重复文本')
const vec2 = await bge.embed('重复文本') // 从缓存读取
```

### 3. 索引优化

**推荐做法**：

```typescript
// 创建 HNSW 索引加速搜索
CREATE INDEX memories_embedding_hnsw_idx
ON memories USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);

// 创建复合索引
CREATE INDEX memories_type_created_idx
ON memories(type, created_at DESC);
```

### 4. Token 压缩

**推荐做法**：

```typescript
// 自动压缩长对话
const tokenWindow = createTokenWindowManager({
  maxTokens: 4000,
})

const { messages: compressed } = await tokenWindow.slideWindow(messages)
// 自动压缩到 4000 tokens 以内
```

## 安全最佳实践

### 1. 环境变量管理

**推荐做法**：

```bash
# .env 文件（不提交到 git）
DATABASE_URL=postgres://user:pass@host:5432/db
BGE_API_KEY=secret-key-here
```

**避免**：

```typescript
// ❌ 不推荐：硬编码
const apiKey = 'xj781102@'
```

### 2. 输入验证

**推荐做法**：

```typescript
function validateInput(input: string): boolean {
  if (input.length > 10000) {
    throw new Error('输入过长')
  }
  if (/[<>"]/.test(input)) {
    throw new Error('输入包含非法字符')
  }
  return true
}

// 使用
if (validateInput(userInput)) {
  await memory.add('user_input', userInput)
}
```

### 3. SQL 注入防护

**推荐做法**：

```typescript
// 使用参数化查询（Drizzle ORM）
const results = await db.select().from(memories).where(eq(memories.id, id)) // ✅ 安全
```

**避免**：

```typescript
// ❌ 不推荐：字符串拼接
const sql = `SELECT * FROM memories WHERE id = '${id}'`
```

## 错误处理

### 1. 优雅降级

**推荐做法**：

```typescript
try {
  const results = await vectorStore.search(embedding, 10)
  return results
} catch (error) {
  // 向量搜索失败，降级到关键词搜索
  console.warn('向量搜索失败，降级到关键词搜索', error)
  return await memory.search('关键词')
}
```

### 2. 重试机制

**推荐做法**：

```typescript
async function retryWithBackoff<T>(fn: () => Promise<T>, maxRetries = 3): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn()
    } catch (error) {
      if (i === maxRetries - 1) throw error
      const delay = Math.pow(2, i) * 1000 // 指数退避
      await new Promise((resolve) => setTimeout(resolve, delay))
    }
  }
  throw new Error('重试失败')
}

// 使用
const result = await retryWithBackoff(() => vectorStore.search(embedding, 10))
```

### 3. 错误日志

**推荐做法**：

```typescript
import pino from 'pino'

const logger = pino()

try {
  await vectorStore.upsert(memory)
} catch (error) {
  logger.error('存储失败', {
    error: error.message,
    memory: memory.type,
    timestamp: new Date(),
  })
  throw error
}
```

## 监控与维护

### 1. 健康检查

**推荐做法**：

```typescript
async function healthCheck() {
  const checks = {
    database: false,
    bge: false,
  }

  try {
    await vectorStore.getStats()
    checks.database = true
  } catch (error) {
    console.error('数据库检查失败', error)
  }

  try {
    await bge.healthCheck()
    checks.bge = true
  } catch (error) {
    console.error('BGE-M3 检查失败', error)
  }

  return checks
}
```

### 2. 性能监控

**推荐做法**：

```typescript
// 监控关键指标
const metrics = {
  searchLatency: [],
  cacheHitRate: 0,
  memoryUsage: 0,

// 记录搜索延迟

// 定期输出
setInterval(() => {
  console.log('性能指标:', metrics)
}, 60000)
```

## 总结

### 关键原则

1. **分层存储**：短期、长期、检查点分离
2. **批量优先**：尽量使用批量操作
3. **缓存策略**：合理使用缓存减少重复计算
4. **安全第一**：环境变量、输入验证、参数化查询
5. **充分测试**：单元、集成、性能测试
6. **优雅降级**：错误时降级而非崩溃
7. **持续监控**：健康检查、性能指标、定期维护

### 常见陷阱

- ❌ 硬编码密钥和配置
- ❌ 逐条操作而非批量
- ❌ 忽略错误处理
- ❌ 不限制缓存大小
- ❌ 不做性能测试
- ❌ 混合职责

# 审查-计算机-根据专利法第25条审查

> **来源：** 《专利审查指南》第二部分第九章 §3、§6.1.2
> **对应法条：** 《专利法》第二十五条第一款第（二）项、第二条第二款
> **最新修订：** 2023年修订（新增AI/大数据相关判断规则）
> **关联页面：** [[审查-计算机-审查基准]]、[[审查-计算机-创造性审查与审查示例]]、[[不授予专利权的客体-智力活动规则与科学发现]]、[[客体-原理-计算程序]]、[[客体-原理-商业方法]]

## 核心要点

涉及计算机程序的发明专利申请首先需要根据专利法第二十五条排除纯智力活动规则，然后根据第二条第二款判断是否构成技术方案。排除审查的关键在于区分"纯粹智力活动规则"和"包含技术特征的混合方案"——前者不授予专利权，后者需进一步审查。2023年修订新增了对AI算法改进与计算机系统内部结构关联的具体判断规则。

---

## 审查标准

### 1. 根据专利法第二十五条第一款第（二）项的审查

#### 1.1 属于智力活动规则的情形

以下类型的权利要求属于智力活动的规则和方法，不属于专利保护的客体：

**【例1】利用计算机程序求解圆周率的方法**

该解决方案仅仅涉及一种由计算机程序执行的纯数学运算方法或者规则，本质属于人的抽象思维方式，属于专利法第二十五条第一款第（二）项规定的智力活动的规则和方法。

**【例2】自动计算动摩擦系数的方法**

该解决方案不是对测量方法的改进，而是一种由计算机程序执行的数值计算方法，求解过程是一种数值计算，整体仍属于一种数学计算方法，属于智力活动的规则和方法。

**【例3】全球语言文字通用转换方法**

该解决方案只是利用统一的翻译中介语，通过人为规定全球语言文字的录入规则，不是对机器翻译方法的改进，没有体现不同语言文字自身固有的客观语言规律与计算机技术结合的改进，本质属于智力活动的规则和方法。

#### 1.2 不排除的情形

如果一项权利要求在对其进行限定的全部内容中既包含智力活动的规则和方法的内容，又包含技术特征，则该权利要求就整体而言并不是一种智力活动的规则和方法，不应当排除其获得专利权的可能性。

### 2. 根据专利法第二条第二款的审查

#### 2.1 属于技术方案的情形

**【例4】控制橡胶模压成型工艺的方法**

利用计算机程序控制橡胶模压成型工艺过程，目的是防止橡胶的过硫化和欠硫化，解决的是技术问题。通过执行计算机程序完成对橡胶模压成型工艺进行的处理，反映的是根据橡胶硫化原理对橡胶硫化时间进行精确、实时控制，属于技术方案。

**【例5】扩充移动计算设备存储容量的方法**

通过虚拟设备文件系统扩充移动计算设备的存储容量，解决的是如何增加移动计算设备有效存储容量的技术问题。通过执行计算机程序实现对移动计算设备内部运行性能的改进，属于技术方案。

**【例6】去除图像噪声的方法**

利用概率统计论中的3θ原理去除图像噪声，解决的是如何在有效去除图像噪声的同时减少图像模糊的问题。通过执行计算机程序实现图像数据的去除噪声处理，利用的是遵循自然规律的技术手段，属于技术方案。

**【例7】测量液体粘度的方法**

通过执行计算机程序对液体粘度测量的数据采集、数据处理和数据显示过程进行自动控制，解决的是如何提高液体粘度测量的速度和精度的技术问题，属于技术方案。

#### 2.2 不属于技术方案的情形

**【例8】计算机游戏方法**

该解决方案通过公知游戏装置执行问答游戏过程控制，但既没有给游戏装置的内部性能带来改进，也没有给游戏装置的构成或功能带来任何技术上的改变。所要解决的问题是如何根据人的主观意志来兼顾两种游戏的特点，不构成技术问题，采用的手段不是技术手段，获得的效果不是技术效果。

### 3. 包含算法特征或商业规则和方法特征的审查（§6.1.2）

#### 3.1 属于技术方案的情形

**【例2】卷积神经网络模型的训练方法**（§6.2）

该解决方案明确了模型训练方法各步骤中处理的数据均为图像数据以及各步骤如何处理图像数据，体现出神经网络训练算法与图像信息处理密切相关。解决的是如何克服CNN模型仅能识别固定尺寸图像的技术问题，属于技术方案。

**【例3】共享单车的使用方法**

该方案通过执行终端设备和服务器上的计算机程序实现了对用户使用共享单车行为的控制和引导，反映的是对位置信息、认证等数据进行采集和计算的控制，利用的是遵循自然规律的技术手段，属于技术方案。

**【例4】区块链节点间通信方法**

该方案解决联盟链网络中如何防止区块链业务节点泄露用户隐私数据的问题，属于提高区块链数据安全性的技术问题，通过CA证书验证限制连接对象，利用的是技术手段，属于技术方案。

**【例5】深度神经网络模型的训练方法**

该方案为解决训练速度慢的问题，针对不同大小的训练数据选择具有不同处理效率的单处理器或多处理器训练方案，与计算机系统的内部结构存在特定技术关联，提升了硬件的执行效果，属于技术方案。

**【例6】电子券使用倾向度的分析方法**

该方案处理的是电子券相关的大数据，挖掘用户行为特征与电子券使用倾向度之间符合自然规律的内在关联关系，解决了提升分析精确性的技术问题，属于技术方案。

**【例7】知识图谱推理方法**

该方案处理的是自然语言中的文本数据或语义信息等技术数据，解决的是文本嵌入及语义搜索过程中如何丰富语义信息、提高推理准确性的技术问题，属于技术方案。

#### 3.2 不属于技术方案的情形

**【例8】消费返利方法**

该方法所要解决的是如何促进用户消费的问题，不构成技术问题。所采用的手段是通过计算机执行人为设定的返利规则，不受自然规律的约束，未利用技术手段。获得的效果仅仅是促进用户消费，不是技术效果。

**【例9】基于用电特征的经济景气指数分析方法**

该方法解决的问题是对经济走势进行判断，不构成技术问题。采用的手段是依照经济学规律采用经济管理手段，不受自然规律的约束，未利用技术手段。

**【例10】金融产品价格预测方法**

该方法利用神经网络模型挖掘金融产品的价格数据之间的关联关系，但金融产品的价格走势遵循经济学规律，历史价格与未来价格之间不存在符合自然规律的内在关联关系，不构成技术问题，获得的不是技术效果。

### 4. 汉字编码方法及计算机汉字输入方法

汉字编码方法属于一种信息表述方法，仅仅涉及汉字编码方法的发明专利申请属于智力活动的规则和方法。

但是，如果把汉字编码方法与该编码方法可使用的特定键盘相结合，构成计算机系统处理汉字的一种计算机汉字输入方法或者计算机汉字信息处理方法，则构成技术方案，属于专利保护的客体。

对于这种由汉字编码方法与特定键盘相结合而构成的计算机汉字输入方法，在说明书及权利要求书中应当描述该汉字输入方法的技术特征，必要时还应当描述该输入方法所使用键盘的技术特征。

## 实务应对

**代理人视角：**

1. **排除审查的应对**：面对第二十五条排除审查，应在权利要求中增加技术特征，使方案整体不属于纯智力活动规则。常见的有效技术特征包括：传感器数据采集、硬件结构改进、通信协议实现等。

2. **技术方案判断的策略**：确保三要素齐全——在背景技术中描述现有技术的技术缺陷（技术问题），在技术方案中描述利用自然规律的具体手段（技术手段），在效果描述中强调符合自然规律的效果（技术效果）。

3. **金融/经济类发明的注意**：金融产品价格预测、经济景气分析等发明通常不被认定为技术方案，因为它们遵循的是经济学规律而非自然规律。撰写时应注意区分技术领域和非技术领域。

4. **大数据分析的撰写要点**：强调数据挖掘的"符合自然规律的内在关联关系"，而非"经济学规律"或"社会学规律"。数据类型应具有确切的技术含义（如图像数据、传感器数据、生物特征数据等）。

5. **区块链发明的撰写**：突出安全性、效率等"技术"效果的提升，而非仅强调商业价值。将共识机制、加密算法等作为技术手段进行描述。

## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 纯数学运算方法（求解圆周率等） | 不授予专利权 | 专利法第二十五条第一款第（二）项 |
| 语言文字转换方法（纯规则） | 不授予专利权 | 人为规定规则，无技术特征 |
| 工业过程控制方法（如橡胶模压控制） | 可授予专利权 | 解决技术问题+技术手段+技术效果 |
| 计算机系统内部性能改进 | 可授予专利权 | 利用自然规律改进系统性能 |
| 技术数据处理方法（如图像去噪） | 可授予专利权 | 处理具有技术含义的数据 |
| 纯游戏方法（无技术改进） | 不授予专利权 | 不构成技术问题，不利用技术手段 |
| CNN模型训练（处理图像数据） | 可授予专利权 | 算法与图像信息处理密切相关 |
| 区块链节点通信（CA证书验证） | 可授予专利权 | 解决数据安全性技术问题 |
| 消费返利方法 | 不授予专利权 | 纯商业规则，不构成技术问题 |
| 金融产品价格预测 | 不授予专利权 | 遵循经济学规律非自然规律 |
| 电子券使用倾向度分析 | 可授予专利权 | 挖掘符合自然规律的内在关联 |
| 汉字编码方法（纯编码规则） | 不授予专利权 | 属于智力活动规则和方法 |
| 汉字编码+特定键盘 | 可授予专利权 | 构成技术方案 |

# 外观设计侵权：图形用户界面（GUI）侵权判定

> **来源：** 《侵权判定指南(2017)理解与适用》第四章，第86-87条
> **核心法条：** 《专利法》第二条第四款、第五十九条第二款；"第68号令"
> **关联页面：** [[外观设计侵权-种类相同与虚实对比]]、[[外观设计侵权-整体观察综合判断]]

---

## 核心要点

图形用户界面（GUI）是我国外观设计保护领域的新事物。静态GUI的侵权判定应主要考虑GUI部分，兼顾其与产品其余部分的位置、比例、分布关系；动态GUI则应重点保护变化过程，比较被诉侵权设计是否能体现出与专利设计一致的变化效果。由于我国尚未建立部分外观设计制度，GUI的保护仍需附着于产品载体，但比对时应弱化硬件产品的影响，突出GUI本身的设计创新。

---

## 第86条 静态图形用户界面

### 条文原文

> 对于静态图形用户界面外观设计，应当主要考虑产品的图形用户界面部分，兼顾其与产品其余部分的关系，如位置、比例、分布关系，与被诉侵权设计中对应的内容进行综合判断。被诉侵权产品的图形用户界面外观设计与专利设计相同或相近似，且与产品其余部分的关系对整体视觉效果不产生显著影响的，应认定被诉侵权设计落入专利权的保护范围。
>
> 被诉侵权设计完整包含了静态图形用户界面外观设计，应认定被诉侵权设计落入专利权的保护范围。

### 理解与适用

**GUI保护的困境：**

1. **保护客体模糊**：通电后显示的图案并非传统意义上的硬件产品固有外观。我国现行制度要求GUI必须以特定工业产品为载体，纯GUI申请不能获得保护。
2. **申请条件僵化**：申请名称须包含产品名称，不能过于上位；不能单独要求保护GUI本身；不允许采用虚实线绘制形式。
3. **保护力度有限**：侵权判定时须考虑产品整体，如果权利人无法证明除GUI外的部分为惯常设计，被告可获较大抗辩空间。

**本条的解决思路：**
- 以GUI部分为准进行比对
- 兼顾GUI与产品其余部分的位置、比例、分布关系
- 当GUI以外的部分为惯常设计时，可认为GUI对整体视觉效果有显著影响

**审查政策的积极变化：**

随着实践发展，审查政策已出现松动：专利名称上位化（如"带图形用户界面的通信设备"）；硬件只需勾勒轮廓；无需提交完整六面视图；简要说明中可强调"设计要点在于屏幕中的GUI内容，硬件是惯常设计"。

### GUI保护范围的界定建议

在暂时不能突破司法解释规定的前提下，应对GUI所附载产品类别作出宽松认定：
- 不受《洛迦诺协定》分类限定
- 若GUI实现了相似的人机交互功能，应视为相近类别产品
- 例如：专利设计在手机上，被诉侵权设计在电脑屏幕、可视电话上，若GUI功能相似，应满足比对前提

## 第87条 动态图形用户界面

### 条文原文

> 对于动态的图形用户界面外观设计，被诉侵权设计与动态的图形用户界面外观设计相比，在图形用户界面的动态变化过程相同或相近似时，应当认定被诉侵权设计落入专利权的保护范围，判断时也要考虑到图形用户界面部分与产品其余部分位置、大小、分布的关系。
>
> 被诉侵权设计缺少部分状态的视图，导致无法体现出与专利设计一致的变化过程的，应当认定被诉侵权设计未落入专利权的保护范围，但仍能唯一确定与专利设计一致的变化过程的除外。
>
> 被诉侵权设计使用了部分动态的图形用户界面外观设计或其关键帧，如果该部分或该关键帧属于图形用户界面外观设计的设计要点，则被诉侵权设计落入专利权的保护范围。但被诉侵权设计整体视觉效果与动态的图形用户界面外观设计不相同且不相近似的除外。

### 理解与适用

**动态GUI的保护重点在于变化过程**，而非某一具体状态。保护范围需结合简要说明对动态变化过程的描述，由能唯一确定动态变化过程的整体产品外观设计视图共同确定。

**动态GUI侵权判定的核心规则：**

1. **变化过程比对**：比较被诉侵权设计的变化能否体现出与专利设计一致的变化效果
2. **关键帧考量**：重点考虑"一个状态的整体产品外观设计视图"和"关键帧"的视图
3. **以下状态对整体视觉效果更具影响**：
   - 整体产品外观设计视图中GUI的状态
   - 关键帧视图中GUI的状态

**缺少部分视图的处理：**
- 导致无法体现一致的变化过程 → 不侵权
- 但仍能唯一确定一致的变化过程 → 不影响侵权认定

**部分使用的处理：**
- 使用了部分动态GUI或关键帧，且属于设计要点 → 原则上侵权
- 但整体视觉效果不相同且不相近似的除外

### 实例说明

**案例一：** 涉案专利"显示面板之图像"为连续动态变化外观设计，被诉侵权对象的变化过程与专利并不相同，变化顺序亦不相同，因二者并未产生混淆的视觉印象，故应认定不相同亦不近似。

**案例二：** 专利号为201430144231.0的"带图形用户界面的手机"，简要说明指出主视图的界面水平滑动后依次呈现界面变化状态图1-2。被诉产品虽不具有如专利中界面变化状态图1所示的"正方体翻转45度"的状态界面，但通过九帧画面完成了翻转的变化过程，能反映出专利所要保护的变化过程，可被认定为相同或者相似。

## GUI侵权判定的实务要点

### 判断步骤

1. **确定产品种类**：以使用该GUI的产品为准
2. **界定保护范围**：结合视图和简要说明，区分GUI部分和硬件部分
3. **举证惯常设计**：若能证明硬件部分为惯常设计，比对重点置于GUI
4. **比对GUI设计**：
   - 静态GUI：比较界面本身的内容及与硬件的关系
   - 动态GUI：比较变化过程是否一致，重点关注关键帧
5. **综合判断**：在整体观察的基础上得出结论

### 常见抗辩与应对

| 被诉侵权人抗辩 | 专利权人应对 |
|-------------|-----------|
| 产品种类不同 | 以用途为标准主张产品类别相近 |
| 硬件外观不同 | 举证硬件为惯常设计 |
| GUI有局部差异 | 从整体视觉效果是否无实质性差异角度分析 |
| 动态过程不同 | 比较是否能体现一致的变化效果 |

## 本页关键概念索引

| 概念 | 含义 |
|------|------|
| GUI（图形用户界面） | Graphical User Interface，通过图形方式实现人与设备交互的界面 |
| 静态GUI | 界面内容固定的图形用户界面 |
| 动态GUI | 界面内容随操作或时间发生变化的图形用户界面 |
| 关键帧 | 动态变化过程中具有代表性的特定状态的视图 |
| 惯常设计 | 现有设计中一般消费者所熟知的、提到产品名称就能想到的相应设计 |
| "第68号令" | 《国家知识产权局关于修改〈专利审查指南〉的决定》，明确了GUI外观设计的审查标准 |

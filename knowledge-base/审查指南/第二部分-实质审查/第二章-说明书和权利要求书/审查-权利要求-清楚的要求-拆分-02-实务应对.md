## 实务应对

### 代理人撰写要点

**1. 权利要求类型的准确表述**

主题名称必须清楚表明权利要求的类型。常见的错误写法包括：
- "一种……技术"（类型模糊）
- "一种……产品及其制造方法"（混合类型）

正确的做法是分别撰写产品权利要求和方法权利要求。

**2. 禁用用语的识别与替代**

审查指南明确列出了权利要求中的禁用用语：
- 含义不确定用语："厚""薄""强""弱""高温""高压"等——应替换为具体数值
- 选择性用语："例如""最好是""尤其是""必要时"——应拆分为不同权利要求
- 模糊用语："约""接近""等""或类似物"——应尽量使用精确表述
- 不必要的括号——应避免使用

**3. 开放式与封闭式表述的选择**

"包含""包括"为开放式表述，保护范围较宽；"由……组成"为封闭式表述，保护范围较窄。选择时应考虑技术方案的实际需要：
- 如果组分的增减不影响发明目的，宜采用开放式
- 如果必须限定为特定组分，宜采用封闭式
- "主要由……组成"介于两者之间，允许含有少量其他成分

**4. 独立权利要求的"划界"技巧**

前序部分只写与发明密切相关的共有必要技术特征，不需要将所有共有特征都写入。特征部分写明区别于最接近现有技术的必要技术特征。划界的目的在于使公众清楚看出哪些是已有特征、哪些是创新特征。

**5. 多项从属权利要求的"禁止嵌套"规则**

这是实务中容易出错的地方。关键规则是：多项从属权利要求不能引用另一项多项从属权利要求。撰写时应建立引用关系图，确保不违反这一规则。

**6. 并列选择概括的等效性要求**

使用"或者"并列的各个选择项应当是等效的。不得将上位概念与其下位概念并列。同时，"类似物"等含义不清的概念不能与具体技术特征并列。

### 审查意见通知书常见表述 → 代理人答复策略对照表

| 审查意见常见表述 | 代理人答复策略 | 援引依据 |
|-----------------|--------------|---------|
| "权利要求中使用'厚''薄'等不确定用语，保护范围不清楚" | 若该用语在特定技术领域具有公认的确切含义，提供教科书/行业标准作为证据；否则替换为精确数值或相对比例 | 第二部分第二章§3.2.2 |
| "权利要求中使用'约''接近'等模糊用语，不清楚" | 论证该用语在所属技术领域属于合理表述（如工艺波动范围）；或修改删除但说明其必要性 | 第二部分第二章§3.2.2 |
| "一项权利要求中包含产品和方法两种主题，类型不清楚" | 将混合主题拆分为产品权利要求和方法权利要求，分别撰写 | 第二部分第二章§3.2.2 |
| "多项从属权利要求引用了另一多项从属权利要求，不符合规定" | 修改引用关系，确保多项从属权利要求不引用另一多项从属权利要求 | 细则第25条第2款 |
| "权利要求中使用了'例如''最好是'，保护范围不清楚" | 将选择性用语拆分至不同权利要求中，或删除选择性用语保留其一 | 第二部分第二章§3.2.2 |

### 具体争辩话术示例

**话术一：证明模糊用语在特定领域有公认含义**
> "审查意见认为权利要求中使用的'[用语]'含义不确定，导致保护范围不清楚。然而，在[所属技术领域]中，'[用语]'具有公认的确切含义，指[具体含义]。例如，[教科书/行业标准/技术手册名称]中明确指出'[用语]是指[定义]'（参见附件[证据编号]）。因此，本领域技术人员能够清楚理解该用语的含义，权利要求的保护范围是清楚的。"

**话术二：区分"不清楚"与"不精确"**
> "审查意见认为权利要求中'[数值范围/用语]'导致保护范围不清楚。申请人认为，'不清楚'与'不精确'是两个不同的概念。权利要求的保护范围只要对于本领域技术人员是可界定的，就是清楚的。本案中，权利要求中的[数值范围/用语]虽然具有一定的概括性，但其上下限/边界对于本领域技术人员是明确的（[说明如何界定]）。如果审查员认为该范围过宽，这属于'是否得到说明书支持'的问题（专利法第26条第4款），而非'保护范围是否清楚'的问题（专利法第26条第4款）。两者适用法律条款不同，不应混淆。"

### 权利要求清楚性答复模板

```
一、关于用语不清楚的反驳

审查意见认为权利要求X中使用的'[用语]'导致保护范围不清楚。申请人认为：

（一）该用语在特定技术领域具有公认含义
在[所属技术领域]中，'[用语]'具有公认的确切含义，指[具体定义]。申请人提交了[教科书/行业标准/技术手册]作为证据（参见附件），证明该用语的含义对于本领域技术人员是清楚的。

（二）即使存在多种理解，最宽理解仍在可接受范围内
即使'[用语]'存在多种理解，按照最宽泛的理解方式，其涵盖的范围仍然[在合理范围内/能够得到说明书的支持]。因此，不会导致保护范围不清楚。

二、关于修改建议（如适用）

若审查员坚持认为'[用语]'不清楚，申请人愿意将其修改为'[修改后的精确表述]'，该修改不会超出原权利要求的保护范围，且能得到说明书的支持。

三、关于多项从属权利要求引用不当的修改（如适用）

权利要求X（多项从属）引用了权利要求Y（多项从属），违反了专利法实施细则第25条第2款的规定。申请人愿意将其修改为'根据权利要求[单选]所述的……'，以消除该缺陷。
```

## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 主题名称含"一种……技术" | 类型不清楚，要求修改 | §3.2.2 |
| 一项权利要求含产品和方法的混合主题 | 类型不清楚，要求拆分 | §3.2.2 |
| 使用"例如""最好是"等选择性用语 | 保护范围不清楚，要求修改 | §3.2.2 |
| 使用"约""接近"等模糊用语 | 视具体情形判断是否清楚 | §3.2.2 |
| 两项以上保护范围实质相同的同类权利要求 | 违反简要要求 | §3.2.3 |
| 多项从属权利要求引用另一多项从属权利要求 | 不允许，要求修改 | 细则25.2 |
| 上位概念与下位概念用"或者"并列 | 并列概括不当 | §3.3 |
| "类似物"与具体特征并列 | 含义不清楚，不允许 | §3.3 |
| 独立权利要求未分前序部分和特征部分 | 开拓性发明等情况下允许 | 细则24.2 |
| 开放式与封闭式混用 | 视具体表述判断保护范围 | §3.3 |

# YunPat 多语言架构决策记录

**日期**: 2026-04-28
**状态**: ✅ 已批准
**决策者**: 项目负责人 + 技术团队

---

## 📋 决策内容

**选择**: TypeScript + Rust + Python（隔离）作为 YunPat 框架的多语言架构

---

## 🎯 决策背景

### 当前问题

- 纯 TypeScript 架构在性能瓶颈（向量检索、任务调度）
- Python ML 工具直接集成导致性能下降
- 无法充分利用多语言优势

### 长期目标

- 支持 5-10 年发展
- 保持高性能（P99 < 10ms）
- 充分利用 AI 生态

---

## 🤔 考虑的方案

### 方案 A: 纯 TypeScript

**优势**:

- ✅ 统一语言，团队协作效率高
- ✅ 前后端统一

**劣势**:

- ❌ 性能瓶颈（单线程、内存占用大）
- ❌ 无法处理高性能场景

**结论**: ❌ 不满足长期性能需求

---

### 方案 B: 纯 Rust

**优势**:

- ✅ 性能极致
- ✅ 内存安全

**劣势**:

- ❌ AI 生态弱
- ❌ 开发效率低
- ❌ 团队学习成本高

**结论**: ❌ 不满足快速迭代需求

---

### 方案 C: 纯 Go

**优势**:

- ✅ 性能优秀
- ✅ 云原生

**劣势**:

- ❌ AI 生态差
- ❌ 泛型弱，代码冗长

**结论**: ❌ 不满足 AI 框架需求

---

### 方案 D: TypeScript + Python（混合）

**优势**:

- ✅ TypeScript 快速迭代
- ✅ Python AI 生态

**劣势**:

- ❌ Python 性能问题
- ❌ GIL 限制并发
- ❌ 内存泄漏风险

**结论**: ❌ Python 会污染核心链路

---

### 方案 E: TypeScript + Rust + Python（隔离）✅

**优势**:

- ✅ TypeScript 快速迭代（编排层）
- ✅ Rust 高性能（核心引擎）
- ✅ Python AI 生态（工具容器）
- ✅ gRPC 接口隔离
- ✅ 长期可持续

**劣势**:

- ⚠️ 多语言维护成本
- ⚠️ gRPC 学习曲线

**缓解措施**:

- Protobuf 接口优先，降低耦合
- 完善的文档和示例
- 渐进式迁移，降低风险

**结论**: ✅ **最佳选择**

## 🎯 架构设计

### 三层架构

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 编排层        │
│  - Agent 协调与事件路由                                 │
│  - HTTP/WebSocket 服务                                  │
│  - 提示词工程与结果聚合                                  │
└────────────────────┬────────────────────────────────────┘
                     │ gRPC/Protobuf
┌────────────────────▼────────────────────────────────────┐
│  Layer 2: 核心引擎层 (Rust - 30%)                      │
│  - 高性能向量检索 (HNSW)                                │
│  - 任务调度与队列管理                                   │
│  - 高性能事件总线                                       │
└────────────────────┬────────────────────────────────────┘
                     │ gRPC/Protobuf
┌────────────────────▼────────────────────────────────────┐
│  Layer 3: 工具提供者层 (Python - 隔离)                 │
│  - ML 模型推理 (PyTorch、TF)                            │
│  - 数据分析 (pandas、numpy)                             │
│  - 容器化，限制资源                                     │
└─────────────────────────────────────────────────────────┘
```

### 语言占比

- **TypeScript**: 70%（编排层）
- **Rust**: 30%（核心引擎）
- **Python**: 0%（完全隔离，仅作工具容器）

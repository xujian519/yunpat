## 任务

基于中国《专利法》第22条第3款（创造性）标准，使用"三步法"框架对专利创造性进行系统评估。

## 核心分析框架：创造性判断三步法

### 第一步：确定最接近的现有技术

**选择标准**（按优先级）：

| 因素 | 说明 | 优先级 |
|------|------|--------|
| 技术领域 | 相同或相近技术领域优先 | 高 |
| 技术问题 | 所解决的技术问题与发明相同或相似 | 高 |
| 发明构思 | 技术思路和技术手段最相似 | 最高 |
| 技术效果 | 产生的技术效果相同或相似 | 中 |

**选择原则**：
1. 技术领域相同或相近优先（如"汽车制动系统" vs "轨道交通制动系统"为相近领域）
2. 所解决的技术问题应当与发明关注的问题有关联
3. **发明构思是最关键因素** — 即使技术领域不同，发明构思相似也可作为最接近现有技术

### 第二步：确定区别特征和实际解决的技术问题

**区别特征确定**：
- 逐一对比发明与最接近现有技术的技术特征，找出不同之处
- 注意：整体考虑、功能对应、层次对应
- 区别特征类型：替代、增加、省略

**实际解决的技术问题确定**：
- 基本方法：区别特征 → 技术效果 → 技术问题
- 关键原则：以区别特征为依据，避免过于上位或过于下位，与发明目的对应

### 第三步：判断是否显而易见（技术启示）

**技术启示判断的五个方面**：

1. **在现有技术中寻找技术手段**：技术手段的作用必须与发明中一致；形式相似不等于技术启示；避免碎片化拼凑
2. **改进动机考量**：改进动机 = 普遍需求 + 公知手段 + 合理成功预期；最接近现有技术教导改进方向则动机更强
3. **发明构思的影响**：发明构思不同 → 形式相似 ≠ 技术启示；不同构思下的判断不能仅看表面相似
4. **准确把握现有技术公开内容**：整体把握不能断章取义；负面描述（"不宜""不适合"）可能反证无技术启示
5. **合乎逻辑的分析推理**：推理链必须严密不跳跃；有限试验可能否定创造性（优选范围内确定最佳值、已知范围内的常规筛选）

## 发明类型判断规则

识别发明类型后应用对应的创造性判断标准：

| 发明类型 | 判断标准 | 具备创造性的条件 |
|----------|---------|-----------------|
| **组合发明** | 各技术特征是否产生协同效应 | 组合产生新功能或效果优于各特征之和 |
| **选择发明** | 选择是否产生预料不到的效果 | 预料不到的效果或长期未被发现的选择 |
| **转用发明** | 跨领域转用是否需要克服技术障碍 | 遥远领域转用或产生预料不到的效果 |
| **要素变更发明** | 变更是否突破常规认识 | 预料不到的效果或突破常规 |
| **要素省略发明** | 省略是否带来预料不到的效果 | 省略后功能不减或带来新效果 |
| **要素关系变更发明** | 变更是否产生新功能/效果 | 产生非显而易见的技术效果 |

## 辅助判断因素

辅助因素**保障**评判结论的准确性，但**不能单独**作为判断创造性的依据：

1. **长期渴望解决的技术难题**：本领域一直渴望解决、长期未获成功、本发明首次解决
2. **技术偏见**：发明申请日之前已存在的普遍认识、本发明克服了该偏见
3. **预料不到的技术效果**：超出技术人员预期、需要实验数据证明（化学/生物领域尤为重要）

## 常见错误警示

| 错误类型 | 表现 | 规避方法 |
|----------|------|---------|
| **事后诸葛亮** | 知晓发明后反向寻找"启示" | 严格基于申请日以前的现有技术客观判断 |
| **碎片化拼凑** | 从多篇现有技术抽离零散特征简单拼凑 | 需要有结合以解决技术问题的启示 |
| **仅看形式相似** | 忽视发明构思和技术手段的实际作用 | 把握发明构思，分析技术手段在不同方案中的作用 |
| **忽视发明构思** | 不考虑发明的整体技术思路 | 发明构思不同时形式相似不等于技术启示 |

## 数据库检索策略

### 相似无效决定检索

在三步法第三步（判断是否显而易见）中，调用 `PostgreSQLClient.queryInvalidDecisions()` 搜索与技术领域和区别特征相似的无效决定，作为创造性分析的佐证。

**检索时机**：第三步分析技术启示时。
**用法**：`queryInvalidDecisions({ query: "技术领域+区别特征描述", topK: 5 })`

### 审查标准检索

在辅助判断因素分析中，调用 `PostgreSQLClient.vectorSearch()` 在法条向量表中搜索相关审查标准。

**检索时机**：分析预料不到的效果、技术偏见等辅助因素时。
**用法**：`vectorSearch("预料不到的技术效果 审查标准", 3)`

### 知识图谱概念推理

在分析技术启示和发明构思时，利用 `PostgreSQLFirstKnowledgeGraph` 进行概念层次推理：
- 查询技术概念的上下位关系，辅助判断技术领域是否相近
- 查询概念间的关联路径，辅助分析技术启示的逻辑链
- 通过 OpenClaw 知识图谱的图遍历，发现隐性技术关联

**检索时机**：确定最接近现有技术和分析发明构思时。

## 评估标准

支持多国创造性标准：

- **cn**：中国（《专利法》第22条第3款，三步法）
- **pct**：PCT 国际检索
- **ep**：欧洲（EPO 问题-解决法）
- **us**：美国（Graham 要素）

## 评估维度

1. **突出实质性特点**：与最接近现有技术的区别特征分析
2. **显著进步**：技术效果对比（定量/定性）
3. **技术贡献**：对现有技术的实际贡献度
4. **组合启示**：多篇对比文件组合是否显而易见
5. **商业成功**：市场认可、许可情况（如有数据）
6. **长期需求**：是否满足长期行业需求

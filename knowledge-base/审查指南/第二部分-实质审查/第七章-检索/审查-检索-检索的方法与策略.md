# 审查-检索-检索的方法与策略

> **来源：** 《专利审查指南》第二部分第七章 §6-12
> **对应法条：** 《专利法》第22条、第9条第1款 / 《专利法实施细则》第54条
> **最新修订：** 无特别修改
> **关联页面：** [[审查-检索-检索的目的与范围]]、[[审查-实质程序-审查意见通知书]]、[[程序-原理-实质审查]]

## 核心要点

检索按照初步检索、常规检索和扩展检索的顺序进行，检索策略包括选择数据库、表达基本检索要素、构建检索式和调整策略。审查员在找到充分对比文件或确认不可能找到时可中止检索。特殊情况下（跨领域、缺乏单一性）有专门的检索规则。检索报告中用X、Y、A、R、P、E、T、L等符号标注对比文件类型。

---

## 审查标准

### 6. 对发明专利申请的检索

#### 6.1 检索的要点

审查员在检索时，要把注意力集中到新颖性上，同时也要注意和创造性有关的现有技术，把那些相互结合后可能使申请的主题不具备创造性的两份或者多份对比文件检索出来。

在检索时，审查员应当注意现有技术中专利文献的全部内容，尤其是专利文献的说明书（及其附图）的内容，而不应仅将注意力放在权利要求书上。

#### 6.2 检索过程

审查员通常根据申请的特点，按照初步检索、常规检索和扩展检索的顺序进行检索。

**6.2.1 初步检索：** 审查员应利用申请人、发明人、优先权等信息检索申请的同族申请、母案/分案申请、申请人或发明人提交的与申请的主题所属相同或相近技术领域的其他申请，还可以利用语义检索，以期快速找到可以对申请的主题的新颖性、创造性有影响的对比文件。

**6.2.2 常规检索：** 常规检索是在申请的主题的所属技术领域进行的检索。所属技术领域是申请的主题所在的主要技术领域，在这些领域中检索，找到密切相关的对比文件的可能性最大。如果通过常规检索发现确定的技术领域不正确，审查员应当重新确定技术领域。

**6.2.3 扩展检索：** 扩展检索是在功能类似或应用类似的技术领域进行的检索。如果在所属技术领域中检索不到对比文件，应当到功能类似或应用类似的技术领域进行扩展检索。

#### 6.3 检索策略

制定检索策略通常包括选择检索系统或数据库、表达基本检索要素、构建检索式和调整检索策略。

**6.3.1 选择检索系统或数据库：** 审查员一般需要考虑申请的主题的所属技术领域、预期要检索文件的国别和年代、检索时拟采用的检索字段和检索系统能够提供的功能、申请人/发明人的特点等因素。

**6.3.2 表达基本检索要素：**
- 基本检索要素的表达形式主要包括：分类号、关键词等
- 对于体现申请的主题的基本检索要素应当优先用分类号进行表达
- 在用关键词表达时，通常首先使用最基本、最准确的关键词，再逐步从形式上（不同词性、单复数、常见拼写错误）、意义上（同义词、近义词、反义词、上下位概念）、角度上（技术问题、技术效果）三个层次完善

**6.3.3 构建检索式：** 审查员可以将同一个基本检索要素的不同表达方式构造成块，结合申请的主题的特点和检索情况，运用逻辑运算符对块进行组合构建检索式。块的组合方式包括全要素组合检索、部分要素组合检索和单要素检索。

**6.3.4 调整检索策略：** 审查员一般需要根据检索结果以及对新颖性和创造性评价的预期方向调整检索策略，包括：
1. 调整基本检索要素的选择
2. 调整检索系统/数据库
3. 调整基本检索要素的表达

#### 6.4 抵触申请的检索

**6.4.1 基本原则：** 在对申请发出授予专利权的通知前，抵触申请的检索应当完成到尽可能完善的程度。

**6.4.2 申请满十八个月公布后进入实质审查程序的检索：** 审查员在发出第一次审查意见通知书之前所作的检索，应当包括抵触申请的检索。

**6.4.3 申请提前公布后进入实质审查程序的检索：** 审查员在发出第一次审查意见通知书之前，可以初步检索抵触申请。如果对该申请作出审查结论的日期在从该申请的申请日起十八个月之内，审查员可视抵触申请进入检索用专利文献中的情况不断进行抵触申请的补充检索；如果作出审查结论的日期在该申请的申请日起满十八个月当天或者之后，审查员应当在满十八个月当天或者之后进一步进行抵触申请检索。

### 7. 防止重复授权的检索

在对申请发出授予专利权的通知前，防止重复授权的检索应当完成到尽可能完善的程度，即应当将在中国专利文献中已经有的涉及同样的发明创造的专利申请或者专利文件检索出来。

### 8. 中止检索

#### 8.1 检索的限度

从成本的合理性角度考虑，检索要有一定的限度。审查员要随时根据已经检索出的对比文件的数量和质量决定是否应当中止检索。考虑的原则是用于检索的时间、精力和成本与预期可能获得的结果要相称。

审查员在没有获得对比文件而决定中止检索时，应当至少在最低限度数据库内进行了检索。最低限度数据库一般情况下应当包括中国专利文摘类数据库、中国专利全文类数据库、外文专利文摘类数据库、英文专利全文类数据库以及中国期刊全文数据库。对于一些特定领域的申请，还应当包括该领域专用数据库。

#### 8.2 可中止检索的几种情况

检索过程中，出现下列情况之一时，审查员可以中止检索：

1. 审查员已经找到一份对比文件，清楚地公开了申请的全部主题的全部技术特征，即构成X类文件或E类文件
2. 审查员已经找到两份或者多份对比文件，结合起来影响申请的全部主题的创造性，即构成Y类文件
3. 审查员根据其知识和工作经验，认为不可能找到密切相关的对比文件，或者认为预期的结果与需要花费的时间、精力和成本相比十分不相称
4. 审查员从公众提供的材料中或者从申请人提交的外国检索资料中，发现了上述（1）或（2）所述的对比文件

### 9. 特殊情况的检索

#### 9.1 申请的主题跨领域时的检索

如果申请的主题跨越不同的技术领域，审查员除了在其负责审查的技术领域进行检索外，还应当视情况与负责其他技术领域的审查员商量，决定如何进行进一步检索。

#### 9.2 申请缺乏单一性时的检索

**9.2.1 对明显缺乏单一性的申请的检索：** 审查员在分析研究权利要求书和说明书后就能判断申请的主题之间缺乏单一性的，可以采取：
1. 待申请人修改并消除缺陷后再进行检索
2. 如果缺乏单一性的权利要求涉及的检索领域非常接近或很大程度上重叠，可以同时完成检索，以加速审查进程

**9.2.2 对不明显缺乏单一性的申请的检索：** 不明显缺乏单一性的申请，是指只有经过检索才能确定缺乏单一性的申请。审查员应当：
1. 对第一独立权利要求进行检索，若发现不具备新颖性或创造性，则根据已有检索结果判断其余独立权利要求之间是否缺乏单一性
2. 如果两项或多项相互并列的独立权利要求在发明构思上非常接近且不需要在其他技术领域检索，可以对全部主题进行检索

### 10. 不必检索的情况

一件申请的全部主题属于下列情形之一的，审查员对该申请不必进行检索：

1. 属于《专利法》第5条或者第25条规定的不授予专利权的情形
2. 不符合《专利法》第2条第2款的规定
3. 不具备实用性
4. 说明书和权利要求书未对该申请的主题作出清楚、完整的说明，以致于所属技术领域的技术人员不能实现

### 11. 补充检索

在申请的实质审查过程中，有下列情形之一的，审查员应当进行补充检索：

1. 申请人修改了权利要求，原先的检索没有覆盖修改后权利要求请求保护的范围
2. 申请人澄清了某些内容，使得原先的检索不完整、不准确
3. 第一次审查意见通知书以前的检索不完整或者不准确
4. 审查意见的改变使得已经作出的检索不完整或者不准确

此外，对于可能构成抵触申请的指定中国的国际专利申请文件，在对申请发出授予专利权的通知之前，应当通过补充检索查看其是否进入了中国国家阶段并作出了中文公布。

### 12. 检索报告

检索报告用于记载检索的结果，特别是记载构成相关现有技术的文件，以及与检索过程有关的检索记录信息。

审查员应当在检索报告中清楚地记载检索到最接近的现有技术的主要检索式，准确列出由检索获得的对比文件以及对比文件与申请主题的相关程度。

#### 对比文件类型符号

| 符号 | 含义 |
|------|------|
| **X** | 单独影响权利要求的新颖性或创造性的文件 |
| **Y** | 与其他Y类文件组合后影响权利要求的创造性的文件 |
| **A** | 背景技术文件 |
| **R** | 同一申请日的同样的发明创造的专利或专利申请文件 |
| **P** | 中间文件（公开日在申请日与优先权日之间） |
| **E** | 抵触申请文件 |
| **T** | 申请日或优先权日当天或之后公布的，可对发明的理论或原理提供解释的文件 |
| **L** | 除上述类型外的原因引用的文件 |

其中，X、Y和A表示对比文件与申请的权利要求在内容上的相关程度；R和E同时表示时间关系和内容相关程度；P表示时间关系，其后应附带标明内容相关程度的符号。

## 实务应对

**代理人实务要点：**

1. **检索报告的分析：** 收到审查意见通知书时，应仔细分析检索报告中对比文件的类型标注。X类文件单独影响新颖性/创造性，Y类文件需与其他文件组合。P类文件涉及优先权问题，需优先核实优先权是否成立。

2. **补充检索的触发条件：** 当申请人修改权利要求后，审查员可能进行补充检索。大幅修改权利要求可能引入新的对比文件风险，实务中应在修改前评估潜在的补充检索结果。

3. **中止检索的启示：** 审查员找到X/Y/E类文件即可中止检索，因此审查意见通知书中引用的对比文件可能不是最全面的。代理人在答辩时也应自行检索，以评估审查员引用的文件是否确实是最接近的现有技术。

4. **抵触申请的时间窗口：** 审查员会在授权前进行尽可能完善的抵触申请检索。提前公布的申请在审查结论作出于18个月之前时，审查员会视情况进行补充检索。实务中应注意可能的抵触申请风险。

5. **不必检索情形的利用：** 当申请存在严重实质性缺陷（如不授予专利权的客体、不具备实用性、公开不充分）时，审查员不必检索即可发出审查意见通知书。此时审查意见可能仅指出主要问题。

6. **E类文件（抵触申请）的重要性：** E类文件单独影响新颖性，无需与其他文件组合。一旦被引用，通常需要修改权利要求或论证不构成抵触申请。抵触申请的认定标准（相同主题、申请日在前、公布日在后）是争辩的焦点。

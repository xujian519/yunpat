## 编译规则

### 1. 读取规则

- 必须完整读取 raw 文件后再编译，禁止只读部分
- OCR 误差处理：根据上下文推断正确内容，常见问题：
  - 条文编号错误：`第 条` → `第65条`
  - 乱码字符：`汨围` → `保护范围`，`霁面` → `界面`
  - 缺字：`专 法` → `专利法`
  - 法条引用缺失数字：`第 条` → `第59条`

### 2. 页面拆分规则

- 按条文分组，每个页面覆盖 1-5 条相关条文
- 单个页面控制在 3000-8000 字
- 如果某条文解析内容超过 8000 字，按子主题拆分
- 页面命名格式：`{章节主题}-{子主题}.md`

### 3. 页面结构模板（严格遵守）

```markdown
# {页面标题}

> **来源：** 《侵权判定指南(2017)理解与适用》第{X}章，第{条文号}条，页{起始}-{结束}
> **核心法条：** 《专利法》第{X}条第{X}款
> **关联页面：** [[{关联页面名}]]

---

```

### 4. 案例提取规则

- 每个案例必须包含五个要素：案件名称、审理法院、争议焦点、裁判要点、启示
- 案例中的法律条文引用必须保留具体法条号
- 启示部分要体现对专利侵权诉讼实务的指导意义
- 如果原文中案例信息不完整，标注 `[信息不完整]`

### 5. 交叉引用规则

- 使用 `[[页面名]]` 格式创建 wiki 链接
- 每个页面必须关联到：同章节的其他页面 + 其他章节的相关页面
- 概念交叉示例：
  - 保护范围确定 ↔ 侵权判定
  - 全面覆盖原则 ↔ 等同侵权
  - 禁止反悔原则 ↔ 权利要求解释
  - 外观设计保护范围 ↔ 外观设计侵权判定
  - 专利侵权抗辩 ↔ 权利保护范围
- 与 `Wiki/专利实务/` 的交叉引用格式：`[[专利实务/页面名]]`

### 6. 输出规则

- 所有文件使用 UTF-8 编码
- 文件名使用中文，不含空格和特殊字符
- 每个页面独立可读，不依赖其他页面的上下文
- 术语首次出现时给出简要解释

# 审查-不授权-智力活动的规则和方法

> **来源：** 《专利审查指南》第二部分第一章 §4.2
> **对应法条：** 《专利法》第二十五条第一款第（二）项
> **最新修订：** 2010年版（沿用至今）
> **关联页面：** [[审查-不授权-违反法律与公序良俗]]、[[审查-计算机-根据专利法第25条审查]]、[[不授予专利权的客体-智力活动规则与科学发现]]、[[客体-原理-智力活动规则]]、[[客体-原理-科学发现]]

## 核心要点

根据《专利法》第二十五条第一款第（二）项的规定，智力活动的规则和方法不授予专利权。智力活动是指人的思维运动，它源于人的思维，经过推理、分析和判断产生出抽象的结果，或者必须经过人的思维运动作为媒介才能间接地作用于自然产生结果。单纯涉及智力活动规则和方法的发明创造（如纯粹的算法、数学计算规则、游戏规则等）不属于专利保护的客体。

---

## 审查标准

### 1. 智力活动的规则和方法的特征

智力活动的规则和方法是指导人们进行思维、表述、判断和记忆的规则和方法。由于其没有采用技术手段或者利用自然规律，也未解决技术问题和产生技术效果，因而不构成技术方案，不能被授予专利权。

### 2. 典型排除情形

以下内容属于智力活动的规则和方法：

（1）**纯粹的算法或数学计算规则**：仅涉及数学运算方法、公式推导等的发明。

（2）**计算机程序本身**：仅仅记录在载体上的计算机程序本身，不包括与硬件结合形成技术方案的情况。

（3）**游戏的规则和方法**：不包括任何技术性特征的纯游戏规则。

（4）**汉字编码方法**：单纯涉及人为规定的编码规则。

（5）**商业规则和方法**：不包含任何技术特征的纯商业模式。

### 3. 不排除的情形

如果一项权利要求在对其进行限定的全部内容中既包含智力活动的规则和方法的内容，又包含技术特征，则该权利要求就整体而言并不是一种智力活动的规则和方法，不应当依据专利法第二十五条排除其获得专利权的可能性。

**关键判断标准：**
- 如果权利要求除主题名称外，限定的全部内容仅仅涉及智力活动的规则和方法，则属于排除范围。
- 如果权利要求中包含了技术特征（如与特定硬件的结合），则不应被排除。

### 4. 智力活动规则与计算机程序发明的边界辨析

#### 4.1 界限划分的核心标准

智力活动规则与计算机程序发明的界限划分，核心在于权利要求是否包含**技术特征**以及是否构成**技术方案**。具体而言：

| 判断维度 | 智力活动规则（不授权） | 计算机程序发明（可授权） |
|---------|-------------------|---------------------|
| 技术手段 | 无技术手段 | 包含技术手段 |
| 自然规律利用 | 不利用自然规律 | 利用自然规律 |
| 技术问题解决 | 未解决技术问题 | 解决了技术问题 |
| 技术效果 | 未产生技术效果 | 产生了技术效果 |
| 硬件结合 | 无硬件结合 | 与硬件结合或控制硬件 |
| 数据处理 | 抽象的数据处理规则 | 针对具体技术领域的数据处理 |

#### 4.2 涉及计算机程序的发明审查原则

涉及计算机程序的发明专利申请属于智力活动规则和方法情形的，按照本部分第一章第4.2节的原则进行审查：

（1）如果一项权利要求仅仅涉及一种算法或数学计算规则，或者计算机程序本身或仅仅记录在载体上的计算机程序本身，或者游戏的规则和方法等，则该权利要求属于智力活动的规则和方法。

（2）如果一项权利要求在对其进行限定的全部内容中既包含智力活动的规则和方法的内容，又包含技术特征，则不应当依据专利法第二十五条排除其获得专利权的可能性。

**【例如】**

**属于智力活动规则的情形：**
- 一种用于股票预测的算法，仅涉及数据处理和数学运算规则；
- 一种计算机程序本身，仅记录在载体上，未与任何硬件或具体应用领域结合；
- 一种游戏的规则，如棋类游戏的胜负判定规则。

**不属于智力活动规则的情形：**
- 一种用于图像处理的算法，与图像采集设备结合，解决了图像清晰度提升的技术问题；
- 一种工业控制程序，与PLC（可编程逻辑控制器）结合，实现了生产线的自动化控制；
- 一种导航方法，与GPS硬件和地图数据库结合，解决了路径规划的技术问题。

#### 4.3 汉字编码方法的特殊规定

汉字编码方法属于一种信息表述方法，仅仅涉及汉字编码方法的发明专利申请属于专利法第二十五条第一款第（二）项规定的智力活动的规则和方法。

但是，如果把汉字编码方法与该编码方法可使用的特定键盘相结合，构成计算机系统处理汉字的一种计算机汉字输入方法或者计算机汉字信息处理方法，使计算机系统能够以汉字信息为指令，运行程序，从而控制或处理外部对象或者内部对象，则这种计算机汉字输入方法构成专利法第二条第二款所说的技术方案，不再属于智力活动的规则和方法，而属于专利保护的客体。

**实务要点：** 单独的汉字编码规则不能获得专利保护，但将编码规则与特定键盘结构结合形成的汉字输入方法可以。代理人在撰写涉及汉字输入的专利申请时，应当在权利要求中明确限定键盘的键位定义、键盘结构等技术特征。

### 5. 商业方法在我国专利审查中的定位演变

#### 5.1 商业方法的定义与排除

商业规则和方法，是指不以技术特征为支撑的、纯粹的商业经营策略、管理方法或交易模式。纯粹的商业规则和方法不包含任何技术特征，属于智力活动的规则和方法，不能被授予专利权。

#### 5.2 定位演变

**早期阶段（2000年以前）：** 我国专利审查实践中对商业方法专利持较为严格的排斥态度，认为商业方法本质上属于人类智力活动，不属于专利保护的客体。

**发展阶段（2000-2010年）：** 随着电子商务和信息技术的快速发展，涉及计算机技术和网络技术的商业方法申请大量涌现。审查实践逐渐认识到，如果商业方法与计算机技术、网络技术等技术手段紧密结合，解决了技术问题并产生了技术效果，则可能构成专利保护的客体。

**现阶段（2010年至今）：** 根据现行《专利审查指南》的规定，涉及商业模式的权利要求，如果既包含商业规则和方法的内容，又包含技术特征，则不应当依据专利法第二十五条排除其获得专利权的可能性。但审查的重点转向该权利要求是否构成专利法第二条第二款规定的"技术方案"。

**审查标准：** 对于涉及商业方法的专利申请，审查员应当判断：
- 权利要求是否包含技术特征；
- 该技术特征是否与技术问题的解决和技术效果的产生相关；
- 权利要求整体是否构成利用技术手段解决技术问题的技术方案。

**【例如】**

**属于排除范围的商业方法：**
- 一种商品定价策略，仅涉及市场分析和定价模型；
- 一种会员积分管理方法，仅涉及商业规则的设计；
- 一种保险理赔流程，仅涉及业务规则的安排。

**可能获得保护的商业方法：**
- 一种基于区块链技术的供应链金融交易系统，利用分布式账本技术解决了交易数据不可篡改和多方信任的技术问题；
- 一种基于大数据分析的个性化推荐方法，利用数据处理技术解决了信息过载的技术问题；
- 一种基于物联网的智能库存管理方法，利用传感器网络和数据分析技术解决了库存实时监控的技术问题。

### 6. 更多审查示例

#### 6.1 算法类发明的审查示例

**【示例1】排除情形——纯数学算法**

权利要求：一种用于求解线性方程组的算法，其特征在于采用迭代方法逐步逼近解。

分析：该权利要求仅涉及数学计算方法，没有与任何具体技术领域或硬件装置结合，未解决技术问题，属于智力活动的规则和方法。

**【示例2】可授权情形——图像处理算法**

权利要求：一种CT图像去噪方法，包括：获取CT扫描原始图像；采用改进的中值滤波算法对图像进行去噪处理；输出处理后的图像。

分析：该权利要求虽然包含算法，但该算法应用于具体的CT图像处理技术领域，解决了医学图像去噪的技术问题，产生了提高图像清晰度的技术效果，属于可被授予专利权的客体。

#### 6.2 游戏规则类发明的审查示例

**【示例3】排除情形——纯游戏规则**

权利要求：一种棋类游戏的胜负判定规则，包括以下步骤：……

分析：该权利要求仅涉及游戏规则的设计，不包含任何技术特征，属于智力活动的规则和方法。

**【示例4】可授权情形——游戏装置**

权利要求：一种虚拟现实游戏交互装置，包括：头戴式显示器、动作捕捉传感器、触觉反馈手套，以及用于根据玩家动作控制游戏场景的处理器。

分析：该权利要求包含具体的硬件装置和技术特征，解决了虚拟现实游戏中人机交互的技术问题，不属于智力活动的规则和方法。

#### 6.3 信息处理方法类发明的审查示例

**【示例5】排除情形——纯信息分类方法**

权利要求：一种图书分类方法，根据图书的主题内容将其归入相应的类别。

分析：该权利要求仅涉及人为规定的分类规则，属于智力活动的规则和方法。

**【示例6】可授权情形——基于技术特征的信息处理**

权利要求：一种基于RFID技术的图书馆智能管理方法，包括：在图书上贴附RFID标签；在书架和出入口布置RFID读写器；通过RFID读写器采集图书位置信息；根据采集的信息进行图书定位和库存管理。

分析：该权利要求利用RFID技术解决了图书馆图书定位和库存管理的技术问题，包含技术特征并产生了技术效果，不属于智力活动的规则和方法。

## 实务应对

**代理人视角：**

1. **技术特征的嵌入**：在撰写涉及算法、商业方法或游戏规则的专利申请时，务必在权利要求中嵌入技术特征。例如，将算法与具体的硬件装置结合、将商业规则与数据处理系统结合，避免权利要求被认定为纯粹的智力活动规则。

2. **算法类发明的撰写策略**：纯粹算法不授予专利权，但如果算法应用于具体的技术领域（如图像处理、信号处理、工业控制），且解决了技术问题，则可以被接受。撰写时应强调：
   - 算法处理的具体技术数据类型；
   - 算法解决的具体技术问题；
   - 算法产生的具体技术效果；
   - 算法与硬件装置或技术系统的结合方式。

3. **汉字编码与输入方法的区分**：单独的汉字编码方法不能获得专利保护，但将编码方法与特定键盘结合的输入方法可以。实务中应在说明书中描述该输入方法的技术特征，包括键位定义和键盘结构等。

4. **商业方法专利的撰写策略**：在中国，纯商业方法不能获得专利保护。撰写时应将商业规则与计算机技术手段紧密结合，描述技术实现的具体方式，使权利要求体现为"利用技术手段解决技术问题的方案"。

   **具体操作建议：**
   - 在权利要求中明确限定技术硬件（如服务器、传感器、网络设备等）；
   - 描述数据处理的技术流程，而非商业逻辑本身；
   - 强调技术方案解决的技术问题（如系统效率、数据安全、通信可靠性等）；
   - 避免使用纯商业术语描述权利要求的实质内容。

5. **答复审查意见时的应对策略**：

   当审查员以"属于智力活动的规则和方法"为由发出审查意见时，代理人可从以下角度争辩：

   （1）**技术特征的存在性：** 指出权利要求中已经包含了技术特征（如硬件装置、技术参数、工艺流程等）；

   （2）**技术方案的构成性：** 论证权利要求整体构成利用技术手段解决技术问题的技术方案；

   （3）**技术效果的说明：** 强调发明产生的技术效果（如提高效率、降低成本、改善性能等），而非仅涉及商业效果或抽象规则；

   （4）**修改策略：** 如果权利要求确实过于抽象，可在权利要求中增加技术特征的限定，如增加与硬件结合的技术方案、限定具体应用领域等。

## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 纯粹的算法或数学计算规则 | 不授予专利权 | 专利法第二十五条第一款第（二）项 |
| 单纯的计算机程序本身 | 不授予专利权 | 属于智力活动的规则和方法 |
| 纯游戏规则和方法 | 不授予专利权 | 无技术特征 |
| 汉字编码方法（纯编码规则） | 不授予专利权 | 人为规定的编码规则 |
| 纯商业规则和方法（无技术特征） | 不授予专利权 | 属于智力活动规则和方法 |
| 算法+技术特征（如工业控制算法） | 可授予专利权 | 包含技术特征，不排除 |
| 汉字编码方法+特定键盘 | 可授予专利权 | 构成技术方案 |
| 游戏规则+技术特征（游戏装置） | 不排除，需进一步审查 | 整体不属智力活动规则 |
| 计算机可读存储介质（仅记录程序） | 不授予专利权 | 实质上仅涉及智力活动规则 |
| 商业模式+技术特征（如区块链交易系统） | 不依据第25条排除，需审查技术方案 | 包含技术特征 |
| 涉及计算机程序的信息处理方法（全部步骤由计算机实施） | 需判断是否为技术方案 | 依据专利法第二条第二款审查 |

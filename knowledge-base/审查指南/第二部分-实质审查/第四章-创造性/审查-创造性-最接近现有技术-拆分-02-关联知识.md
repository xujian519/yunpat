## 审查要点速查

| 情形 | 审查结论 | 关键依据 |
|------|---------|---------|
| 技术领域相同、技术问题相关联的现有技术 | 应优先选为最接近的现有技术 | §3.2.1.1(1)——应首先考虑技术领域相同或相近，优先考虑与技术问题相关联的 |
| 技术领域相同但与发明技术问题无关联 | 不应优先选取 | §3.2.1.1(1)——优先考虑与发明要解决的技术问题相关联的现有技术 |
| 技术领域不同但公开技术特征最多且能实现发明功能 | 可选为最接近的现有技术 | §3.2.1.1(1)——跨领域但功能实现且公开特征最多 |
| 仅公开技术特征多但技术领域和技术问题均不相关 | 不宜作为最接近现有技术 | §3.2.1.1(1)——应首先考虑技术领域，技术问题关联是核心过滤条件 |
| 多份同领域对比文件均可用 | 选技术问题最接近或技术特征公开最多的 | §3.2.1.1(1)——综合考量技术问题、技术效果、用途及公开特征数量 |
| 技术领域相近但发明构思根本不同、技术手段相悖 | 不适合作为最接近现有技术 | §3.2.1.1(1)——"最密切相关"要求整体关联性，非单一特征比较 |
| 跨领域对比文件功能可实现且公开特征最多 | 可作为最接近现有技术，但需严格审查两条件 | §3.2.1.1(1)——跨领域选择的例外 |
| 对比文件虽公开部分特征但技术问题完全无关 | 不应作为最接近现有技术 | §3.2.1.1(1)——技术问题关联性决定改进动机 |

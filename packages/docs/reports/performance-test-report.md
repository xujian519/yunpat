## 🔍 问题分析

### 发现的问题

#### 1. PostgreSQL 查询失败

**症状**：

- 结构化查询返回 0 结果
- 向量搜索报错：`tsquery` 语法错误
- 只有 YunPat 核心概念在工作

**原因分析**：

1. **向量嵌入未生成**：`patent_decisions_v2_embeddings` 表可能为空
2. **表结构不匹配**：PostgreSQL 查询使用的列名与实际表结构不符
3. **中文分词问题**：`to_tsquery` 对中文查询处理有问题

**错误示例**：

```
tsquery syntax error: "发明名称：基于深度学习的图像识别方法..."
```

#### 2. 知识覆盖率低

**症状**：

- prior-art-search: 0% (0/4)
- invention: 0% (0/4)

**原因**：

- PostgreSQL 查询失败导致无法检索法律条文和无效决定
- 只有 YunPat 核心概念在工作，但覆盖范围有限

#### 3. 查询成功率低

**症状**：

- 5个查询中只有2个返回结果
- 成功率仅 40%

**原因**：

- YunPat 核心概念只覆盖部分关键词
- PostgreSQL 查询完全失败

## 💡 解决方案

### 短期方案（1-2天）

#### 1. 生成向量嵌入

**优先级**：🔴 高

**方案**：运行向量嵌入生成脚本

```bash
# 使用 Python 脚本生成向量嵌入
python3 scripts/generate_invalid_decision_embeddings.py
```

**预期效果**：

- 为 9,562 条无效决定生成向量嵌入
- 启用向量相似度搜索
- 提升查询成功率到 80%+

#### 2. 修复中文查询

**优先级**：🔴 高

**方案**：使用简单的 `LIKE` 查询替代全文搜索

```typescript
// Before: 使用 to_tsquery (不支持中文)
WHERE to_tsvector(content) @@ to_tsquery('深度学习')

// After: 使用 LIKE (支持中文)
WHERE content LIKE '%深度学习%'
```

**预期效果**：

- 解决中文查询报错
- 提升查询稳定性

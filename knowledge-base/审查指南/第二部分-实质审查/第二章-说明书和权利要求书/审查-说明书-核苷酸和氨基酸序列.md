# 审查-说明书-核苷酸和氨基酸序列

> **来源：** 《专利审查指南》第二部分第二章 细则第20条第4款、第一部分第一章第4.2节、第二部分第十章第9.2.3节

## 核心要点

发明专利申请涉及核苷酸或氨基酸序列时，说明书必须包含符合规定的序列表（Sequence Listing），且序列表应作为说明书的单独部分提交。序列表既是满足专利法第二十六条第三款充分公开要求的重要技术载体，也是权利要求中获得序列限定特征说明书支持的必要基础。电子申请与纸件申请的提交形式要求不同，但当计算机可读形式与书面记载不一致时，均以书面提交的序列表为准。

---

## 审查标准

### 一、序列表的法定要求（第二部分第二章 细则20.4）

**细则 20.4** 发明专利申请包含一个或者多个核苷酸或者氨基酸序列的，说明书应当包括符合规定的序列表。有关序列表的提交参见本指南第一部分第一章第 4.2 节。

该条款明确了序列表提交义务的触发条件——"包含一个或者多个核苷酸或者氨基酸序列"，且将序列表作为说明书的必要组成部分。未提交符合规定的序列表，可能导致说明书公开不充分的缺陷，进而构成专利法实施细则第五十九条规定的驳回情形。

> **【审查员视角】**
> 在涉及序列表的审查中，审查员对"形式瑕疵"与"实质公开不充分"的界限把握不一。部分审查员将未提交计算机可读形式序列表副本、序列表格式不符合ST.26标准等形式问题，直接等同于说明书公开不充分（法26.3），发出驳回倾向的审查意见；而根据指南规定，这类形式问题通常应先发出补正通知书，只有在确实因缺少序列信息导致技术人员无法实现发明时，才构成公开不充分。另一种常见偏差是：审查员在发现计算机可读形式序列表与书面序列表存在个别字符差异时，直接认定"不一致"并要求补正，而不考虑该差异是否影响序列的实质含义。代理人的有效应对策略是：首先区分审查员指出的问题是形式补正（如格式、副本缺失）还是实质公开不充分——对于前者，应在指定期限内及时补正；对于后者，应评估书面说明书中是否已通过文字方式充分描述了序列信息，若是，可争辩即使缺少序列表形式文件，说明书文字描述已满足法26.3的要求。此外，在PCT国际申请进入国家阶段时，审查员有时会对国际阶段提交的序列表格式提出质疑，代理人应提前准备符合国家知识产权局要求的ST.26/XML格式序列表，避免因形式转换问题延误审查进程。

### 二、序列表的提交形式与格式要求（第一部分第一章第4.2节）

涉及核苷酸或者氨基酸序列的申请，应当将该序列表作为说明书的一个单独部分。审查指南对电子申请和纸件申请分别规定了不同的提交要求：

**（一）电子申请**

对于电子申请，应当提交一份符合规定的计算机可读形式序列表作为说明书的一个单独部分。电子申请中的序列表应与说明书文字部分记载的内容保持一致。

**（二）纸件申请**

对于纸件申请，应当提交单独编写页码的序列表，并且在申请的同时提交与该序列表相一致的计算机可读形式序列表的副本，如提交记载有该序列表的符合规定的光盘或者软盘。

**（三）不一致的处理规则**

提交的光盘或者软盘中记载的序列表与说明书中的序列表不一致的，以说明书中的序列表为准。未提交计算机可读形式序列表的副本，或者所提交的副本与说明书中的序列表明显不一致的，审查员应当发出补正通知书，通知申请人在指定期限内补交正确的副本。期满未补交的，审查员应当发出视为撤回通知书。

### 三、序列表的一致性审查

**计算机可读形式与书面记载不一致时的处理**

如果申请人提交的计算机可读形式的核苷酸或者氨基酸序列表与说明书和权利要求书中书面记载的序列表不一致，则以书面提交的序列表为准。这一规则意味着：

1. 书面序列表具有优先法律效力；
2. 代理人和申请人在制作序列表时，必须确保计算机可读形式与书面记载的一致性；
3. 审查员在审查时，若发现不一致，将以书面文本为审查基础，并可能要求申请人澄清或补正。

### 四、生物技术领域序列表的特殊标准（第二部分第十章第9.2.3节）

第二部分第十章对生物技术领域发明专利申请的序列表提出了更为具体的技术要求：

**（一）序列长度的阈值标准**

细则 20.4 （1）当发明涉及由 **10 个或者更多核苷酸**组成的核苷酸序列，或者由 **4 个或者更多 L-氨基酸**组成的蛋白质或者肽的氨基酸序列时，应当递交符合国家知识产权局规定的序列表电子文件。

该阈值是判断是否需要提交序列表的客观标准：
- **核苷酸序列**：长度 ≥ 10 个核苷酸（nucleotide）
- **氨基酸序列**：长度 ≥ 4 个 L-氨基酸（L-amino acid）

低于上述阈值的短序列，不强制要求以序列表形式提交，但仍应在说明书中以普通文字清楚描述。

**（二）序列表作为说明书单独部分的地位**

细则 20.4 （2）序列表应作为说明书的一个单独部分。有关序列表的提交参见第一部分第一章第 4.2 节。

"作为说明书的一个单独部分"意味着序列表在形式上独立于说明书的"技术领域、背景技术、发明内容、附图说明、具体实施方式"等常规章节，通常单独编排页码，并在说明书目录或开头予以说明。

在生物技术领域，权利要求中涉及基因、多肽或蛋白质的发明时，序列表与权利要求的关联尤为紧密：

**（一）基因的限定方式**

**（二）多肽或蛋白质的限定方式**

### 六、实质审查程序中的序列表审查（第二部分第八章）

在实质审查程序中，审查员应当对包含核苷酸或氨基酸序列的申请进行以下审查：

专利申请包含一个或多个核苷酸或氨基酸序列的，应当审查说明书是否包括符合规定的序列表。

审查员需要核实：
1. 申请是否确实包含核苷酸或氨基酸序列（达到阈值标准的）；
2. 说明书是否包含序列表；
3. 序列表的格式是否符合规定（电子文件是否为规定的计算机可读形式）；
4. 计算机可读形式副本与书面序列表是否一致；
5. 权利要求中引用的SEQ ID NO是否在序列表中存在且一致。

## 实务应对

### 一、撰写与提交前的准备

**1. 判断是否需要提交序列表**

在撰写涉及生物技术的发明专利申请时，首先应当核查权利要求和说明书中是否记载了以下序列：
- 长度达到或超过 10 个核苷酸的 DNA、RNA 等核酸序列；
- 长度达到或超过 4 个 L-氨基酸的蛋白质、多肽序列。

若存在上述序列，则必须准备并提交序列表。注意：即使权利要求中未直接限定序列，但说明书中公开了相关序列以支持发明的充分公开，也应提交序列表。

**2. 序列表的电子文件格式**

目前，国家知识产权局要求序列表电子文件符合 WIPO ST.26 标准（XML 格式）。ST.26 标准（WIPO Standard ST.26）是世界知识产权组织制定的关于核苷酸和氨基酸序列表呈列的标准，采用 XML（可扩展标记语言）格式，替代了此前的 ST.25 标准（纯文本格式）。实务中应注意：
- 确认当前官方接受的序列表标准版本（目前为 ST.26/XML）；
- 使用官方认可的工具（如 WIPO Sequence Suite、PatentIn 等）生成序列表；
- 电子文件扩展名和格式应符合受理要求。

**3. 确保书面序列表与电子文件完全一致**

由于审查规则明确规定"不一致时以书面提交的序列表为准"，代理人和申请人应当建立双重核对机制：
- 在提交前，将书面序列表（PDF 或纸件）与计算机可读形式（XML 文件）进行逐条比对；
- 重点核对序列总数、每个序列的 SEQ ID NO、序列长度、具体碱基/氨基酸残基、修饰标记等；
- 避免因软件导出错误或手动修改导致两者不一致。

### 二、序列表内容的准确性

**1. 序列编号（SEQ ID NO）的连续性**

序列表中的序列应当按 SEQ ID NO:1、SEQ ID NO:2……的顺序连续编号，不得跳号或重号。说明书和权利要求中引用序列时，应确保引用的 SEQ ID NO 在序列表中存在。

**2. 序列与说明书描述的一致性**

序列表中记载的序列必须与说明书具体实施方式中的描述完全一致。例如：
- 说明书中描述的"编码酶 A 的基因"，其碱基序列在序列表中必须准确对应；
- 权利要求中限定的"在（a）限定的氨基酸序列中经过取代、缺失或添加一个或几个氨基酸……的蛋白质"，其基础序列（a）必须在序列表中有明确记载。

**3. 不完整序列的处理**

如果说明书中描述的序列由于技术原因（如仅测定了部分序列）无法完整提供，应在说明书中充分解释原因，并评估是否影响充分公开。若完整序列对于实现发明是必不可少的，则未提供完整序列表可能导致公开不充分。

### 三、审查意见答复策略

**1. 收到补正通知书（未提交计算机可读形式副本或副本不一致）**

应当在指定期限内补交正确的计算机可读形式序列表副本。注意：
- 补交的副本必须与说明书中的书面序列表完全一致；
- 若书面序列表本身存在错误，应同时提交对说明书的改正，以确保一致性；
- 期满未补正将导致视为撤回，务必关注期限。

**2. 收到公开不充分审查意见（序列表缺失或不完整）**

若审查员认为未提交序列表导致说明书公开不充分，申请人可以：
- 在允许修改的范围内补交序列表（需确保不超出原说明书和权利要求书记载的范围，否则违反专利法第三十三条）；
- 若原申请文件中已记载了序列的文字描述但未以序列表形式提交，可争辩说明该文字描述已满足充分公开要求，同时补交序列表作为形式补正；
- 若原申请文件中确实未记载关键序列信息，则补交序列表可能因超范围而不被接受，此时需评估是否可以通过生物保藏信息等其他方式补救。

**3. 权利要求得不到说明书支持（序列表与权利要求不一致）**

若权利要求中引用了序列表中不存在的 SEQ ID NO，或引用的序列与权利要求限定不符，应当：
- 修改权利要求中的 SEQ ID NO 引用，使其与序列表一致；或
- 修改序列表（但需注意不得超范围，通常只能删除错误或补充原已记载的序列）。

### 四、国际申请与分案申请的特殊注意

**1. PCT 国际申请进入国家阶段**

PCT 国际申请进入中国国家阶段时，若国际阶段提交的序列表不符合国家知识产权局规定的格式，可能需要在进入国家阶段时重新提交符合规定的序列表。应关注国际公布文本中的序列表信息与国家阶段要求的差异。

**2. 分案申请中的序列表**

分案申请应当保留原申请的序列表内容。若分案申请的权利要求涉及了原申请序列表中已有的序列，通常无需重新提交；但若分案申请新增了序列限定，需确保新增内容不超范围，并视情况补交序列表。
